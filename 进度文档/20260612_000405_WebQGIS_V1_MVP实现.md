# WebQGIS V1 MVP 实现进度

## 本次目标

根据 `WebGIS_QGIS_like_V1_plan.md` 搭建类 QGIS WebGIS V1 的可运行 MVP 骨架，实现 PostGIS 数据源连接、空间表扫描、图层注册、矢量瓦片发布地址、要素回源读取与编辑保存接口，并提供一个 Vue + OpenLayers 前端用于图层浏览、选择、绘制、节点编辑、吸附、属性编辑和保存/删除。

## 做了哪些

- 新建 npm workspaces 项目结构，包含 `apps/api` 后端和 `apps/web` 前端。
- 后端使用 Fastify + pg 实现：
  - PostGIS 连接测试。
  - 数据源本地保存与脱敏列表。
  - `geometry_columns` 空间表扫描。
  - schema、table、geometry column、SRID、几何类型、单字段主键、字段、权限、空间索引、范围元数据识别。
  - 图层注册表生成，可编辑状态和不可编辑原因判断。
  - MVT 矢量瓦片接口，使用 `ST_TileEnvelope`、`ST_AsMVTGeom`、`ST_AsMVT`。
  - 按主键回源读取原始 GeoJSON 要素。
  - 要素新增、更新、删除接口，写回 PostGIS 原始表。
- 前端使用 Vue 3 + OpenLayers 实现：
  - 数据源表单、保存和扫描入口。
  - 图层树、图层显示开关、活动图层切换、只读/可编辑状态展示。
  - MVT 图层加载。
  - 点击地图要素后按主键回源读取原始几何与属性。
  - 编辑草稿图层、点/线/面绘制、Modify 节点编辑、Snap 吸附。
  - 属性面板和保存/删除/清空草稿操作。
- 新增 README，说明启动方式、接口和 V1 边界。
- 新增 `.gitignore`，忽略依赖、构建产物、运行时数据和环境文件。
- 已执行验证：
  - `npm install` 成功。
  - `npm run typecheck` 成功。
  - `npm run build` 成功。
  - 启动 `apps/api/dist/server.js` 后访问 `GET /api/health` 成功。

## 没做哪些

- 未接入真实示例 PostGIS 数据库做端到端业务数据联调。
- 未实现用户登录、角色权限体系和连接配置加密保存。
- 未实现属性字段高级约束、枚举、日期控件、精度控制和只读规则细分。
- 未实现撤销/重做、编辑冲突检测、乐观锁和事务级批量保存。
- 未实现高级 QGIS 样式、标注、图例、打印制图、空间分析和文件导入导出。
- 未实现服务端全库吸附、拓扑规则和多用户协同编辑。
- 未加入自动化测试用例，当前以类型检查、生产构建和 API 健康检查为验证依据。

## 哪里有 bug 或风险

- 当前尚未用真实 PostGIS 表验证全部 SQL 路径，MVT、扫描 extent、CRUD 在不同 PostGIS 版本或复杂表结构下可能还需要调整。
- 数据源密码当前保存到本地 `data/datasources.json`，只适合本地 MVP，不适合生产环境。
- 前端选择 MVT 要素后依赖瓦片属性里的 `id` 字段，若图层无主键或瓦片要素未携带 id，则只能展示不能回源编辑。
- 新增要素时如果目标表有非空且无默认值字段，保存会由数据库报错，前端暂未提供字段约束预校验。
- 几何合法性目前依赖数据库写入报错，尚未显式调用 `ST_IsValid` 返回友好错误。
- 当前吸附只对编辑草稿源生效，未把当前视野内业务图层要素加载成专用吸附源。

## 建议后续做什么

- 准备示例 PostGIS 数据库，包含点、线、面三张表，每张表都有主键、SRID、空间索引和基础属性字段，跑通完整演示链路。
- 增加后端集成测试，覆盖数据源扫描、只读图层判定、MVT 输出、要素读取和 CRUD。
- 对数据源密码加密存储，并补充多用户权限隔离。
- 增加 `ST_IsValid`、字段类型转换、非空字段和数据库错误码到中文提示的映射。
- 为编辑流程补撤销/重做、保存前确认、保存失败保留草稿、乐观锁版本字段支持。
- 将当前视野内可见要素加载为吸附源，实现真正的“当前视野内客户端吸附”。
