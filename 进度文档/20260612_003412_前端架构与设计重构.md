# 前端架构与设计重构进度

## 本次目标

根据用户反馈“当前前端代码以及设计有问题，不该这样写”，结合已点名的 Vue、Vue Best Practices、VueUse、Vitest、Vue Testing、Vite、UnoCSS、Web Design Guidelines 等要求，对 WebQGIS 前端进行结构化重构：拆分单文件大组件、明确组件职责与数据流、改善 GIS 工作台界面可维护性与可访问性，并补充前端测试和构建验证。

## 做了哪些

- 将原本全部挤在 `App.vue` 的 API 状态、地图交互、图层树、数据源表单、属性编辑面板拆分为更清晰的结构：
  - `App.vue` 只负责挂载工作台。
  - `WebGisWorkbench.vue` 作为页面组合容器。
  - `DatasourcePanel.vue` 管理数据源 UI。
  - `LayerPanel.vue` 管理图层树 UI。
  - `MapCanvas.vue` 管理地图画布与编辑工具栏 UI。
  - `EditInspector.vue` 管理图层元数据和属性编辑 UI。
- 新增 `useWebGisWorkspace.ts`，集中管理数据源、图层、活动图层、可见图层、选中要素、草稿几何、状态消息和 API 调用。
- 新增 `useOpenLayersEditor.ts`，集中管理 OpenLayers 地图实例、MVT 图层挂载、选择、绘制、Modify、Snap、草稿加载、图层刷新和删除确认流程。
- 新增 `types/gis.ts`，把 GIS 前端类型从 API 文件中抽出，避免组件和业务逻辑互相耦合。
- 新增 `utils/layer.ts`，沉淀可测试的纯逻辑：可编辑字段过滤、图层状态文案、数值字段识别、绘制类型约束。
- 新增 `utils/layer.test.ts`，使用 Vitest 覆盖上述纯逻辑。
- 接入 VueUse：
  - `useTitle` 设置工作台标题。
  - `useConfirmDialog` 管理删除要素确认流程，避免直接执行破坏性操作。
- 接入 UnoCSS：
  - 新增 `uno.config.ts`。
  - 在 `main.ts` 引入 `virtual:uno.css`。
  - 定义少量通用 shortcut 和主题 token，当前仍以组件 scoped CSS 为主，避免一次性大规模迁移样式。
- 优化设计与可访问性：
  - 数据源表单补充显式 label、name、autocomplete、inputmode、min/max。
  - 地图工具栏保留固定尺寸与响应式换行，避免文字和按钮挤压。
  - 删除操作改为确认弹窗。
  - 状态消息增加 success/warning/danger 语义。
  - 全局 CSS 收缩为 reset 和基础 token，旧的 `.shell/.panel/.toolbar` 等临时 Demo 样式已删除。
- 已执行验证：
  - `npm install` 成功。
  - `npm run typecheck` 成功。
  - `npm --workspace apps/web run test` 成功，1 个测试文件、4 个测试通过。
  - `npm run build` 成功。

## 没做哪些

- 未引入 Vue Router。当前仍是单工作台页面，没有多页面路由需求，强行引入会增加不必要复杂度。
- 未引入 VitePress。当前任务是应用前端重构，不是文档站建设。
- 未引入 tsdown。当前项目是 Web 应用和 API 服务，不是需要独立发布的 TypeScript 库。
- 未把所有样式迁移成 UnoCSS 原子类。当前采取渐进式策略，先配置 UnoCSS 并保留组件 scoped CSS，避免一次重构改变过多视觉细节。
- 未做真实 PostGIS 数据库端到端 UI 操作验证，因为当前环境没有提供示例数据库连接信息。
- 未做浏览器截图/交互自动化测试，本次验证以类型检查、单元测试、生产构建为主。

## 哪里有 bug 或风险

- `useOpenLayersEditor` 已拆分地图副作用，但仍需要真实 MVT/PostGIS 数据联调确认点击选择、瓦片 feature id、回源读取的完整链路。
- 当前删除确认弹窗是基础实现，尚未加入键盘焦点陷阱和 Esc 关闭。
- 当前属性编辑输入仍是基础 text/number 控件，日期、布尔、枚举、JSON、只读字段提示等控件还未细化。
- UnoCSS 已接入但使用较少，后续如果团队决定全面采用，需要统一样式策略，避免 scoped CSS 与 utility class 长期混杂。
- `vue-tsc` 和 Vitest/Vite 在受限沙箱下会写临时文件失败，本次已通过批准后非沙箱重跑成功。

## 建议后续做什么

- 准备示例 PostGIS 数据库，做一次从扫描图层、加载 MVT、点击查询、编辑草稿、保存刷新到删除确认的端到端验证。
- 为 `WebGisWorkbench.vue` 增加黑盒组件测试，mock API 和地图 composable，验证主流程按钮状态与事件流。
- 为删除确认弹窗补焦点管理、Esc 关闭、点击遮罩关闭策略，并增加测试。
- 继续把地图相关命令抽象成更小的 action，例如 `selectFeature`、`applyDraft`、`syncTileLayers`，降低 OpenLayers composable 的体积。
- 设计系统进一步收敛：决定 UnoCSS 原子类优先还是 scoped CSS 优先，并形成一套按钮、输入框、状态徽标的基础组件。
