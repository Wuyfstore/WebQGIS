# QGIS 风格前端重构进度文档

## 本次目标

基于 `设计稿/WebQGIS_QGIS参考工作台设计.png` 和 `设计稿/WebQGIS_QGIS参考工作台设计.svg`，把前端工作台从偏网页卡片式界面调整为更接近 QGIS 桌面应用的视觉与排版，包括菜单栏、工具栏、停靠面板、图层树、属性表单和底部状态栏。

## 做了哪些

- 将主工作台改为 QGIS 桌面式结构：顶部菜单栏、图标工具栏、上下文工具条、中间三栏工作区、底部状态栏。
- 将左侧数据源区域改为“浏览器”停靠面板样式，包含过滤输入、PostGIS 连接树和紧凑连接表单。
- 将图层区域改为 QGIS 图层树样式，使用符号色块、可见性复选框、选中行和只读/可编辑状态。
- 将图层样式编辑改为独立停靠面板，保留填充色、边线色、透明度、线宽调整能力。
- 将地图画布背景改成设计稿中的浅米色网格底色，编辑工具浮层改为桌面停靠面板质感。
- 将右侧编辑面板改为“图层与要素属性 / 属性表单 / 保存校验 / SQL 审计预览”四段停靠面板。
- 增加全局 QGIS 风格 CSS 变量，统一灰色桌面、边框、输入框、状态色和地图底色。
- 保留原有前端数据流：数据源保存/扫描、图层选择/显隐、样式保存、绘制、保存、删除等事件接口未改。

## 没做哪些

- 没有新增业务 API 或修改后端逻辑。
- 没有实现真实 QGIS 图标库，只用 CSS 绘制了轻量工具栏图标。
- 没有实现可拖拽停靠面板、可折叠面板、图层组排序等高级桌面交互。
- 没有接入真实 PostGIS 示例库做端到端地图数据验证。
- 没有解决 Vite 构建时已有的 chunk 超过 500KB 提示。

## 哪里有 bug

- 本次代码验证未发现阻塞性 bug。
- Browser 插件验证受当前 Windows 自动化环境限制失败，错误为 `CreateProcessAsUserW failed: 5`，因此未能通过 in-app Browser 截图检查。
- 已用构建、类型检查、单元测试和静态产物 HTTP 访问替代验证。
- 已知非阻塞问题：`npm run build` 成功，但仍提示前端 chunk 超过 500KB。

## 建议后续做什么

- 接入真实 QGIS 风格图标或统一 SVG 图标资产，替换当前 CSS 简化图标。
- 增加停靠面板折叠、拖拽调整宽度、图层树分组和排序。
- 增加真实示例 PostGIS 数据源，做地图加载、选择、编辑、保存的端到端视觉验收。
- 对 OpenLayers 相关代码做动态 import 或 manualChunks，降低 Vite chunk 体积警告。
- 将前端 QGIS 风格 tokens 抽成更系统的设计变量和组件规范。
