# 前端错误日志上报进度文档

## 本次目标

- 项目进入页面后用户侧仍然报错，但当前环境无法直接稳定打开浏览器观察错误。
- 按用户要求，在后端增加日志功能，让前端运行时错误可以上报到后端，便于下一次复现后定位真实报错。

## 做了哪些

- 后端新增 `client-logs` 功能模块。
- 新增 `POST /api/client-logs`，接收前端运行时错误日志，返回 `202`。
- 新增 `GET /api/client-logs?limit=...`，查询最近前端日志。
- 使用现有 `JsonFileStore` 将日志写入 `data/client-logs.json`，最多保留 200 条；该文件已被 `.gitignore` 忽略，不进入 Git。
- 对日志 DTO 增加 `class-validator` 校验，限制来源、消息、堆栈、组件名、页面 URL、UA 等字段长度。
- 前端入口安装全局错误上报：
  - Vue `app.config.errorHandler`
  - `window.error`
  - `unhandledrejection`
- 前端优先使用 `navigator.sendBeacon("/api/client-logs")` 上报，失败后退到 `fetch` + `keepalive`。
- 新增后端单元测试，覆盖日志创建、查询条数限制和非法 `limit` 默认值。
- 移除了本次 smoke test 产生的临时文件 `api-log-out.txt`、`api-log-err.txt`。

## 没做哪些

- 未直接定位用户浏览器里“项目刚进入”的真实报错原因，因为当前自动化浏览器/桌面观察此前不可用。
- 未修改现有 WebGIS 业务交互逻辑，本次只增加错误采集链路。
- 未处理生产构建中已有的 Vite 大 chunk 提醒。

## 哪里有 bug 或风险

- 本次验证时，默认 `4100` 端口曾被已有旧进程占用，新启动的 API 进程报 `EADDRINUSE`；随后改用 `API_PORT=4110` smoke test，日志创建和查询通过。
- Vite 构建仍提示 `assets/index-*.js` 超过 500 kB，这是构建警告，不阻塞本次功能。
- 如果用户页面错误发生在前端脚本加载极早期、早于 `main.ts` 安装日志监听之前，仍可能捕获不到。

## 建议后续做什么

- 用户重新进入项目复现报错后，调用 `GET /api/client-logs?limit=20` 查看真实前端错误。
- 在页面上增加开发态“查看最近前端日志”的入口，减少手动查接口的成本。
- 后续可把日志扩展为结构化服务端日志，支持按时间、页面、来源筛选，并避免长期使用本地 JSON 存储。
- 针对 Vite 大 chunk 警告，后续可按 OpenLayers、业务代码做动态拆包。
