# 首屏后端 500 修复进度文档

## 本次目标

- 查看首屏报错日志并排查用户进入项目后仍显示错误的问题。
- 修复本地开发模式下首屏请求 `/api/datasources`、`/api/layers` 返回 500，页面状态栏显示 `Internal server error` 的问题。

## 做了哪些

- 读取 `data/client-logs.json`，确认其中只有手工 smoke 日志，没有真实浏览器运行时错误。
- 使用浏览器复现首屏，确认页面并非 Vue 渲染崩溃，而是状态栏显示后端接口错误。
- 验证当前开发后端 `/api/datasources`、`/api/layers`、`/api/client-logs` 均返回 500。
- 用源码 `tsx src/main.ts` 启动临时 API 端口复现并捕获异常栈：
  - `DatasourcesController.list` 中 `datasourcesService` 为 `undefined`。
  - `LayersController.list` 中 `layersService` 为 `undefined`。
  - `ClientLogsController.list` 中 `clientLogsService` 为 `undefined`。
- 根因判断为开发模式 `tsx` 运行源码时不能依赖构造器参数元数据完成 Nest 依赖注入。
- 为以下构造器补充显式 `@Inject(...)`：
  - `DatasourcesController -> DatasourcesService`
  - `LayersController -> LayersService`
  - `ClientLogsController -> ClientLogsService`
  - `DatasourcesRepository -> JsonFileStore`
  - `LayersRepository -> JsonFileStore`
  - `ClientLogsRepository -> JsonFileStore`
- 清理本次排查残留的空日志文件：`api-dev-out.log`、`api-dev-err.log`、`web-dev-out.log`、`web-dev-err.log`。

## 没做哪些

- 未修改前端业务逻辑，因为前端只是正确展示了后端接口错误。
- 未处理 Vite 构建的大 chunk 警告。
- 未纳入本次任务外的旧改动 `进度文档/20260612_005557_README中文化.md`。

## 哪里有 bug 或风险

- 当前修复覆盖了已发现的 controller 和 repository 注入点；如果后续新增 Nest provider 并继续用 `tsx` 直接跑源码，也应继续使用显式 `@Inject(...)`。
- 浏览器验证时出现过 Codex 浏览器工具自身的 Statsig 网络超时日志，不属于 WebQGIS 应用错误。
- Vite 仍提示构建产物 `assets/index-*.js` 超过 500 kB，这是既有性能警告，不影响本次首屏 500 修复。

## 建议后续做什么

- 在 API 层增加 e2e/smoke 测试，覆盖源码开发模式下 `/api/datasources`、`/api/layers`、`/api/client-logs` 的 200 响应。
- 后续新增 Nest controller/service/repository 时，统一显式使用 `@Inject(...)`，避免开发模式再次出现 DI 元数据缺失。
- 另开任务处理 Vite 大 chunk 警告，可按 OpenLayers 和业务代码做拆包。
