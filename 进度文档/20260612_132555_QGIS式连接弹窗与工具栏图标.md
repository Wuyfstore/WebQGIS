# QGIS 式连接弹窗与工具栏图标进度文档

## 本次目标

- 对比 QGIS 的初步交互，避免 PostGIS 连接表单固定堆在左侧面板中。
- 实现 PostgreSQL 浏览器节点右键新建连接，并弹出 PostGIS 连接窗口。
- 替换顶部快捷工具栏中原先手写 CSS 图标，改用开源图标组件。

## 做了哪些

- 新增 `@element-plus/icons-vue` 依赖，用 Element Plus 开源 SVG 图标替换顶部快捷工具栏图标。
- 将 `DatasourcePanel` 的固定 PostGIS 表单改为弹窗：
  - 左侧浏览器树保留 PostgreSQL 根节点。
  - 右键 PostgreSQL 节点显示上下文菜单。
  - 菜单包含 `新建连接...`、`刷新`、`连接管理`。
  - 点击 `新建连接...` 弹出 `创建新的 PostGIS 连接` 对话框。
  - 弹窗内保留名称、主机、端口、数据库、用户名、密码、SSL、保存并测试能力。
- `DatasourcePanel` 表单改为 `v-model:form` 双向契约，父组件仍使用原有 `datasourceForm` 和 `saveDatasource` 流程。
- 删除顶部工具栏旧 CSS 伪元素图标样式，避免手写图标继续影响显示。
- 增加 WebGisWorkbench 组件测试，覆盖：
  - 右键 PostgreSQL 节点打开上下文菜单。
  - 点击新建连接后显示 PostGIS 连接弹窗。
  - 顶部快捷工具栏渲染 SVG 图标组件。

## 没做哪些

- 未实现完整 QGIS Browser 的全部右键菜单，例如重命名连接、删除连接、编辑连接参数、展开 schema/table 结构。
- 未实现菜单栏中所有按钮的真实业务动作；本次只处理用户明确指出的 PostGIS 新建连接入口和工具栏图标。
- 未处理构建产物超过 500 kB 的 Vite 警告。
- 未纳入本次任务外的旧改动 `进度文档/20260612_005557_README中文化.md`。

## 哪里有 bug 或风险

- 当前 `连接管理` 菜单项暂时复用打开连接弹窗，后续应扩展为编辑/删除已有连接的管理窗口。
- 新增 Element Plus 图标依赖后，生产 JS chunk 从约 508 kB 增加到约 524 kB，Vite 大 chunk 警告仍存在。
- 右键菜单目前只覆盖 PostgreSQL 根节点，未覆盖已有连接节点的专属右键菜单。

## 建议后续做什么

- 为已有连接节点增加 QGIS 风格右键菜单：编辑连接、删除连接、刷新、展开 schema、扫描图层。
- 将 PostGIS 连接弹窗拆成可复用组件，后续支持新建和编辑同一个表单。
- 针对 Element 图标和 OpenLayers 做构建拆包，降低首屏 JS chunk。
- 继续补齐 QGIS 基础操作按钮的真实行为，例如选择、平移、识别、节点编辑、吸附开关。
