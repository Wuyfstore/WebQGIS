# QGIS 工具栏可用化与 PostgreSQL 连接弹窗进度文档

## 本次目标

- 对比 QGIS 当前工作台交互，把顶部大量只有视觉效果、无法使用的按钮改为可触发当前已有能力的工具按钮。
- PostgreSQL 浏览器节点需要支持右键新增连接，点击后出现连接弹窗。
- 保持现有 WebQGIS 架构，不引入新的大范围重构。

## 做了哪些

- 将顶部 QGIS 风格工具栏改为由计算状态驱动：
  - 新建：清空当前草稿并提示开始采集。
  - 保存：保存当前编辑草稿。
  - 撤销：清空当前草稿和选中要素。
  - 选择：启用选择工具。
  - 平移：切换平移工具状态并给出状态栏反馈。
  - 缩放：触发地图放大。
  - 识别：启用识别工具，点击要素后反馈识别结果。
  - 点、线、面：根据当前活动图层几何类型和可编辑状态决定是否启用，并调用绘制能力。
  - 节点：启用节点编辑，并在没有草稿时给出提示。
  - 吸附：切换吸附开关，并同步按钮激活态。
  - 校验：检查当前图层是否可编辑并输出状态栏结果。
  - 刷新：刷新数据源和图层，并刷新当前地图图层。
- 菜单栏按钮不再完全静默，点击后会在状态栏显示当前菜单可用能力或尚未启用的说明。
- PostgreSQL 浏览器根节点增强：
  - 保留右键菜单。
  - 右键菜单包含 `新建连接...`、`刷新`、`连接管理`。
  - 点击 `新建连接...` 会打开 `创建新的 PostGIS 连接` 弹窗。
  - 单击 PostgreSQL 根节点也会直接打开新建连接弹窗，提高可发现性。
  - PostgreSQL 根节点增加当前连接数量提示。
- `useOpenLayersEditor` 提供工具栏所需的地图工具状态和动作：
  - `activeTool`
  - `isSnapEnabled`
  - `activateTool`
  - `toggleSnap`
  - 支持 `startDrawing(mode)` 按按钮直接启动点、线、面绘制。
- 更新组件测试：
  - 覆盖右键 PostgreSQL 节点打开上下文菜单。
  - 覆盖单击 PostgreSQL 节点打开连接弹窗。
  - 覆盖工具栏图标渲染。
  - 覆盖工具栏按钮会调用已有地图工具动作。
- 完成真实浏览器验证：
  - `http://127.0.0.1:5173/` 可打开。
  - PostgreSQL 单击弹窗可见。
  - PostgreSQL 右键菜单可见。
  - 选择、吸附工具可触发，状态栏会显示 `吸附已关闭` 等反馈。
  - 浏览器 console 无 error/warn。

## 没做哪些

- 未实现完整 QGIS 菜单栏的全部业务功能，例如项目保存到文件、插件系统、网络数据源管理等。
- 未实现已有 PostgreSQL 连接节点的完整右键菜单，例如编辑连接、删除连接、重命名、展开 schema/table 树。
- 未新增后端连接测试 API；当前弹窗仍沿用既有 `saveDatasource` 提交流程。
- 未处理 Vite 构建时超过 500 kB chunk 的警告。
- 未处理本次任务外的既有脏改动 `进度文档/20260612_005557_README中文化.md`。

## 哪里有 bug

- `连接管理` 菜单项目前仍复用打开连接弹窗，尚不是真正的连接管理窗口。
- 平移工具目前主要是工具状态和状态栏反馈，OpenLayers 默认拖拽地图仍可用，但尚未做更细的 QGIS 式互斥交互控制。
- 缩放按钮当前执行放大一级，尚未实现框选缩放、缩小、缩放到图层等完整 QGIS 行为。
- 构建仍有 Vite 大 chunk 警告：`assets/index-*.js` 超过 500 kB。

## 建议后续做什么

- 为 PostgreSQL 已有连接节点补齐 QGIS 风格右键菜单：编辑连接、删除连接、刷新、展开 schema、扫描图层。
- 将 PostGIS 连接弹窗拆成独立组件，支持新建和编辑共用一套表单。
- 继续完善 QGIS 工具栏能力：缩小、框选缩放、缩放到图层、属性识别面板、选择清除、保存项目。
- 将菜单栏升级为真正的菜单/下拉命令，而不是只显示状态栏说明。
- 对 OpenLayers、Element Plus 图标等依赖做拆包，降低首屏 JS chunk 体积。
