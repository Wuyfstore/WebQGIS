# 顶部菜单可用化与 PostgreSQL 树节点进度文档

## 本次目标

- WebQGIS 顶部第一行菜单按钮此前基本没有实际作用，需要结合 QGIS 桌面菜单形态和 WebQGIS V1 需求进行修改。
- PostgreSQL 左键不应直接弹出连接窗口，而应只做树状结构展开/闭合。
- PostgreSQL 新增连接仍通过右键菜单或数据库菜单命令触发弹窗。

## 做了哪些

- 将顶部第一行从单纯状态提示按钮升级为 QGIS 风格下拉命令菜单：
  - `项目`：新建编辑草稿、保存当前编辑、刷新工作区。
  - `编辑`：选择要素、撤销当前草稿、节点编辑、删除选中要素。
  - `视图`：平移地图、放大地图、刷新当前图层。
  - `图层`：刷新图层列表、校验当前图层、切换当前图层可见性。
  - `设置`：切换吸附、校验编辑状态。
  - `插件`：明确提示插件系统尚未纳入 V1。
  - `矢量`：绘制点、绘制线、绘制面，并根据当前图层几何类型和可编辑状态禁用不可用命令。
  - `数据库`：新建 PostgreSQL 连接、刷新数据源、扫描第一个数据源。
  - `网络`：刷新 MVT 图层、显示展示链路和编辑链路说明。
  - `帮助`：显示 V1 范围和当前状态。
- `DatasourcePanel` 新增 PostgreSQL 根节点展开状态：
  - 左键只在 `▾ PostgreSQL` 和 `▸ PostgreSQL` 之间切换。
  - 折叠后隐藏已有连接列表和空状态提示。
  - 左键不会打开 PostGIS 连接弹窗。
- 保留 PostgreSQL 根节点右键菜单：
  - `新建连接...` 打开 `创建新的 PostGIS 连接` 弹窗。
  - `刷新` 扫描第一个数据源。
  - `连接管理` 暂时仍复用连接弹窗。
- 增加从顶部 `数据库 > 新建 PostgreSQL 连接...` 打开同一连接弹窗的能力。
- 新增 `connectionDialogRequestKey` 父子组件契约，让父级菜单可以显式请求数据源面板打开连接弹窗。
- 更新前端组件测试：
  - 覆盖 PostgreSQL 左键只展开/折叠且不弹窗。
  - 覆盖 PostgreSQL 右键新建连接弹窗。
  - 覆盖顶部数据库菜单命令打开连接弹窗。
  - 保留工具栏图标和工具栏动作测试。

## 没做哪些

- 未实现完整 QGIS 菜单体系中的所有命令，例如项目文件保存、打印制图、插件安装、复杂矢量分析等。
- 未实现菜单快捷键、键盘方向键导航、点击外部自动关闭所有菜单等完整桌面菜单交互。
- 未实现已有 PostgreSQL 连接节点的编辑、删除、重命名、schema/table 展开树。
- 未实现真正的连接管理窗口；`连接管理` 当前仍复用新建连接弹窗。
- 未处理 Vite 构建时超过 500 kB chunk 的警告。
- 未处理本次任务外的既有脏改动 `进度文档/20260612_005557_README中文化.md`。

## 哪里有 bug

- `连接管理` 不是独立管理界面，仍是临时入口。
- 顶部菜单已具备命令能力，但还不是完整桌面级菜单控件，暂不支持键盘菜单导航和点击空白处关闭。
- 平移、缩放等命令仍是 V1 需要范围内的基础能力，尚未达到 QGIS 完整工具行为，例如框选缩放、缩放到选中、缩放到图层。
- 构建仍有 Vite 大 chunk 警告：生产 JS chunk 超过 500 kB。

## 建议后续做什么

- 为顶部菜单补充键盘可访问性和点击外部关闭逻辑。
- 将数据库连接弹窗拆为独立组件，同时支持新建、编辑、测试连接、删除连接。
- 为 PostgreSQL 已有连接节点增加右键菜单：编辑连接、删除连接、刷新、展开 schema/table、扫描指定连接。
- 为 `项目` 菜单补齐项目保存/恢复工作区配置。
- 对 OpenLayers 和 Element Plus 图标做拆包，降低首屏 JS chunk。
