# 矢量瓦片要素编辑修复进度

## 本次目标

- 修复“使用矢量瓦片服务后，地图上的要素无法进入编辑”的问题。
- 让 MVT 瓦片要素能够携带可识别主键，前端点击要素后可以回源读取原始 PostGIS GeoJSON，再进入节点/属性编辑流程。
- 按要求记录本次进度，并提交、推送到 `main` 分支。

## 做了哪些

- 后端 MVT 输出增强：
  - 在 `ST_AsMVT` 中为整型主键图层输出标准 MVT feature id。
  - 同时保留 `id` 属性作为兼容兜底。
  - 增加 `int2`、`int4`、`int8` 字段类型判断，避免非整型主键误传入 MVT feature id 参数。
- 前端 OpenLayers 编辑逻辑增强：
  - 新增 `readVectorTileFeaturePk`，优先读取 OpenLayers 暴露的 MVT feature id，读取不到时回退到 `id` 属性。
  - VectorTile MVT 解析时使用图层主键作为 `idProperty`。
  - 点击选择矢量瓦片要素后，根据 MVT 的 `layer` 属性定位真实图层，而不是只依赖当前活动图层。
  - 回源读取接口改为 `readFeature(layerId, pk)`，避免多图层场景下读错图层。
  - 读取原始要素成功后加载为可编辑 GeoJSON，并更新选中状态与状态栏提示。
  - 地图初始中心调整到中国范围附近，便于当前测试数据可见。
- 测试补充：
  - 新增 `useOpenLayersEditor.test.ts`，覆盖 MVT feature id、`id` 属性兜底、缺失主键三种情况。
- 本地运行态数据处理：
  - 发现 `apps/api/data/*.json` 包含本地测试数据库连接信息，因此新增 `.gitignore` 规则，避免明文连接信息进入仓库。
