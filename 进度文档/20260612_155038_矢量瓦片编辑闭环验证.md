# 矢量瓦片编辑闭环验证进度

## 本次目标

- 继续完成“用完矢量瓦片服务后无法编辑”的目标。
- 证明 MVT 浏览链路可点击选择要素，前端能回源读取原始 PostGIS geometry，进入可编辑草稿，并能保存写回。
- 修复验证过程中发现的新建绘制草稿误沿用旧选中要素的问题。

## 做了哪些

- 用浏览器实际验证 MVT 图层渲染：
  - 页面已加载 `public.china_boundaries`。
  - OpenLayers 地图内存在 `canvasCount = 1`、`layerDivs = 1`，说明 MVT 图层已真实渲染。
- 用浏览器实际验证点击 MVT 要素后进入编辑：
  - 点击地图要素后，状态栏显示 `已回源读取要素 3271，可进行节点编辑或属性编辑`。
  - 右侧属性面板显示 `要素 3271`。
  - 保存按钮可用。
- 用浏览器实际验证保存写回：
  - 点击保存后，状态栏显示 `保存成功，图层已刷新`。
  - 浏览器控制台无 error/warn。
- 修复绘制草稿相关问题：
  - 移除无用的 OpenLayers `Select` interaction，统一使用 `singleclick + forEachFeatureAtPixel` 处理 MVT 要素选择。
  - 修复 `Draw` 已自动写入 source 后又手动 `addFeature` 导致的重复添加异常。
  - 绘制新要素前清空 workspace 选中状态，避免新绘制草稿沿用旧 `selectedFeatureId`，从而误保存到旧要素。
  - 绘制结束后延后一帧更新草稿 geometry 再切到节点编辑，状态栏能正确显示 `节点编辑已启用：拖动节点修改草稿`。

## 没做哪些

- 没有提交旧进度文档 `进度文档/20260612_005557_README中文化.md` 的无关改动。

- 没有新增 Playwright E2E 文件；本次使用内置浏览器完成了人工自动化验证。

## 哪里有 bug

- 已修复：绘制新面时不再触发 OpenLayers `The passed feature was already added to the source` 异常。
- 已修复：新绘制草稿不再沿用旧要素 ID；验证结果中 `selectedText = null`、删除按钮不可用、保存按钮可用。
- 残留体验问题：右侧 SQL / 审计预览在新建草稿时仍展示 `UPDATE ... WHERE id = ?`，建议后续根据是否选中要素切换 INSERT/UPDATE 预览。

## 建议后续做什么

- 后续做真实几何修改保存验证时，可新增测试专用表或事务回滚，避免污染业务测试数据。
