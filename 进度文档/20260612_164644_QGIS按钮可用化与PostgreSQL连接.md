# QGIS 按钮可用化与 PostgreSQL 连接进度文档

## 本次目标

- 对比 QGIS 桌面端当前需要，把 WebQGIS 顶部大量仅展示、无法操作的按钮改为可触发已有业务能力的命令。
- PostgreSQL 浏览器节点需要支持右键新增连接，点击 `新建连接...` 后出现连接弹窗。
- 后续根据 QGIS 交互习惯调整 PostgreSQL 左键行为：左键用于展开/折叠树节点，新增连接保留在右键菜单和数据库菜单中。
- 任务完成后生成本进度文档，并执行 `git status`、`git add`、`git commit`、`git push origin main`。

## 做了哪些

- 完成 QGIS 风格工具栏可用化：
  - 新建、保存、撤销、选择、平移、缩放、识别、点/线/面绘制、节点编辑、吸附、校验、刷新等按钮已接入当前已有能力或状态栏反馈。
  - 工具按钮根据 `busy`、草稿几何、选中要素、当前活动图层是否可编辑、几何类型等状态动态禁用。
  - `useOpenLayersEditor` 增加 `activeTool`、`isSnapEnabled`、`activateTool`、`toggleSnap` 等工具栏需要的状态和动作。
- 完成 PostgreSQL 根节点右键新增连接：
  - PostgreSQL 根节点支持右键菜单。
  - 右键菜单包含 `新建连接...`、`刷新`、`连接管理`。
  - 点击 `新建连接...` 会打开 `创建新的 PostGIS 连接` 弹窗。
- 完成顶部菜单命令化：
  - `项目`、`编辑`、`视图`、`图层`、`设置`、`插件`、`矢量`、`数据库`、`网络`、`帮助` 已从静态按钮改为 QGIS 风格下拉命令。
  - `数据库 > 新建 PostgreSQL 连接...` 可以打开同一个 PostGIS 连接弹窗。
  - `数据库 > 扫描第一个数据源` 会在有数据源时触发扫描。
- 调整 PostgreSQL 左键交互：
  - 左键只展开/折叠 PostgreSQL 树节点，不再直接弹出连接窗口。
  - 连接弹窗入口集中在 PostgreSQL 右键菜单和顶部数据库菜单。
- 补充前端组件测试：
  - 覆盖 PostgreSQL 右键菜单打开连接弹窗。
  - 覆盖 PostgreSQL 左键只展开/折叠且不弹窗。
  - 覆盖顶部数据库菜单打开连接弹窗。
  - 覆盖工具栏图标渲染和工具栏动作接线。
- 已完成并推送的相关功能提交：
  - `e9517a8 完善QGIS工具栏交互和PostgreSQL连接弹窗`
  - `50ad42c 完善顶部菜单命令和PostgreSQL树交互`

## 没做哪些

- 未实现完整 QGIS 桌面菜单体系的全部命令，例如项目文件保存/打开、打印制图、插件安装、网络数据源管理、复杂矢量分析等。
- 未实现已有 PostgreSQL 连接节点的完整树状管理能力，例如编辑连接、删除连接、重命名、展开 schema/table、扫描指定连接。
- 未实现真正独立的连接管理窗口；当前 `连接管理` 仍复用连接弹窗。
- 未新增后端连接测试 API；当前弹窗仍沿用已有 `saveDatasource` 提交流程。
- 未处理 Vite 构建时生产 JS chunk 超过 500 kB 的警告。
- 未处理本次任务外的既有脏改动 `进度文档/20260612_005557_README中文化.md`。

## 哪里有 bug

- `连接管理` 菜单项目前不是独立管理界面，容易让用户误以为可以管理已有连接。
- 顶部菜单已可触发命令，但还不是完整桌面级菜单控件，暂不支持键盘方向键导航、完善的焦点循环和点击空白处统一关闭。
- 平移、缩放等地图工具仍是 V1 基础行为，尚未达到 QGIS 的完整工具体验，例如框选缩放、缩放到选中、缩放到图层。
- 多个中国边界/省市县图层同时显示时，视觉上容易互相覆盖，选择目标不够清晰。
- 构建仍存在 Vite 大 chunk 警告，后续需要拆分 OpenLayers、Element Plus 图标等依赖。

## 建议后续做什么

- 将 PostGIS 连接弹窗拆成独立组件，同时支持新建、编辑、测试连接、删除连接。
- 为 PostgreSQL 已有连接节点补齐右键菜单：编辑连接、删除连接、刷新、展开 schema/table、扫描指定连接。
- 为顶部菜单补充键盘可访问性、点击外部关闭和更完整的 disabled 原因提示。
- 继续完善 QGIS 工具栏：缩小、框选缩放、缩放到图层、缩放到选中、清除选择、属性识别面板、保存项目。
- 增加“只显示当前图层/快速关闭其他图层”的操作，降低多图层重叠时的判断成本。
- 对 OpenLayers、Element Plus 图标和业务模块做拆包，降低首屏 JS chunk 体积。
