# 20260612_172035_属性表字段面板

## 本次目标

- 复核当前 WebQGIS 中大量按钮不可用的问题，优先对照 QGIS 补齐当前阶段需要真实可用的入口。
- 确认 PostgreSQL 浏览器节点支持右键新增连接，点击后出现连接弹窗。
- 将图层右键菜单中的“打开属性表”从占位提示改为可打开的第一版属性表面板。

## 做了哪些

- 新增 `AttributeTablePanel.vue`，提供 QGIS 风格的紧凑属性表弹窗。
- 属性表面板展示当前图层的 schema/table、字段数量、几何类型、几何字段、SRID、主键、范围、查询/新增/更新/删除/空间索引能力。
- 属性表面板用语义化 table 展示字段清单，包含字段名、类型、可空、默认值、可编辑。
- `WebGisWorkbench.vue` 新增属性表打开/关闭状态，图层右键“打开属性表”会激活图层并展示真实面板。
- 更新组件测试，覆盖右键图层菜单打开属性表、字段显示、状态栏反馈和关闭面板。
- 复核 PostgreSQL 浏览器右键菜单：右键 PostgreSQL 节点可见“新建连接...”，点击后显示“创建新的 PostGIS 连接”弹窗；该路径已有测试覆盖。
- 执行浏览器验证：在 `http://127.0.0.1:5173` 打开页面，右键图层，点击“打开属性表”，确认真实字段显示并可关闭。

## 没做哪些

- 没有拉取和展示真实要素行数据。
- 没有实现属性表分页、筛选、排序、字段搜索。
- 没有实现属性表行选择与地图要素选择联动。
- 没有实现属性表内编辑字段值并写回 PostGIS。
- 没有处理全量工具栏按钮，本次只把当前用户路径中仍是占位的属性表入口补成可用面板，并复核 PostgreSQL 右键新增连接。

## 哪里有 bug

- 本次未发现新增功能性 bug。
- `npm --workspace apps/web run build` 通过，但 Vite 仍提示生产包中 `assets/index-*.js` 超过 500 kB；这是体积优化 warning，不影响本次功能运行。
- 仓库中存在一个与本次无关的未提交文件：`进度文档/20260612_005557_README中文化.md`，本次不会暂存或提交它。

## 建议后续做什么

- 为属性表接入后端要素查询接口，支持分页加载真实 feature rows。
- 增加字段级筛选、排序、列宽调整和字段搜索。
- 实现属性表行选择与地图高亮/缩放联动。
- 将属性表编辑能力和现有 `EditInspector` 的保存链路整合，形成 QGIS 风格的属性编辑流程。
- 对 Vite 大 chunk 做代码分割，例如按 OpenLayers、属性表等模块拆分动态导入。
- 继续梳理工具栏和菜单中仍为占位提示的按钮，按当前 V1 需求逐步改成真实行为。
