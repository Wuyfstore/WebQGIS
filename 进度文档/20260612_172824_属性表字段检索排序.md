# 20260612_172824_属性表字段检索排序

## 本次目标

- 依据上一份进度文档的后续建议，继续增强属性表面板的字段浏览能力。
- 在不扩大到后端要素分页的前提下，先补齐字段过滤和字段列排序，让属性表面板更接近 QGIS 的可浏览体验。

## 做了哪些

- 在 `AttributeTablePanel.vue` 中新增字段过滤输入框，支持按字段名、字段类型、默认值、可空状态、可编辑状态过滤字段。
- 字段数量显示支持过滤态，例如 `1/3 个字段`，无匹配时显示 `无匹配字段`。
- 属性表表头改为可点击排序按钮，支持按字段名、类型、可空、默认值、可编辑排序。
- 使用 VueUse 的 `useSorted` 处理排序结果，避免直接修改图层字段源数据。
- 为表头排序按钮补充 `aria-label`，排序方向通过 `▲` / `▼` 标记显示。
- 更新 `WebGisWorkbench.test.ts`，覆盖字段过滤、无匹配状态和按类型排序后的字段顺序。
- 执行浏览器验证：打开图层右键属性表，输入字段过滤关键字，确认过滤数量和字段结果；重新打开属性表后点击类型表头确认排序标记和字段顺序变化。

## 没做哪些

- 没有实现真实要素行查询、分页、排序或筛选。
- 没有实现属性表行选择与地图高亮联动。
- 没有实现列宽拖拽调整。
- 没有实现字段值编辑和写回 PostGIS。
- 没有处理仍为 V1 外的插件系统入口。

## 建议后续做什么

- 为属性表增加后端要素列表接口，支持按页加载真实 feature rows。
- 在要素行接入后，把字段过滤和排序扩展到真实要素表的列筛选/排序交互。
- 增加属性表行点击选择，并与地图选中、高亮、缩放联动。
- 增加列宽调整与字段可见性设置，便于浏览字段较多的图层。
- 继续处理 Vite 大 chunk warning，优先拆分 OpenLayers 和较少使用的面板模块。
