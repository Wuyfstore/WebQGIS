# 20260615_200048_QGIS操作逻辑设计稿

## 本次目标

结合 QGIS 桌面端 UI 排版与操作逻辑，为当前 WebQGIS 生成一份可落地参考的界面设计稿，重点覆盖：

- QGIS 风格的菜单栏、多排工具栏、停靠面板、地图画布和状态栏布局。
- PostgreSQL/PostGIS 数据源在浏览器面板中的树形管理逻辑。
- PostgreSQL 支持右键新增连接、刷新、连接管理等入口的交互表达。
- 图层右键菜单、图层可见性、缩放到图层、属性表与字段浏览等操作路径。

## 做了哪些

- 读取并遵循本轮显式指定的 `computer-use` 与 `imagegen` 技能说明。
- 按 Computer Use 技能指定入口尝试连接 Windows 应用运行时，用于打开/观察 QGIS。
- 基于当前项目方向与 QGIS 常见工作台结构，完成一份确定性的高保真 SVG 设计稿。
- 使用本地 Chrome headless 将 SVG 导出为 PNG，便于直接预览和在文档/讨论中引用。
- 生成一份可直接用于后续 imagegen 的 UI mockup prompt，便于在 imagegen 工具可用时继续生成模型位图稿。
- 设计稿中体现了以下 UI 与操作逻辑：
  - 顶部菜单：项目、编辑、视图、图层、设置、插件、矢量、数据库、网络、帮助。
  - 多排紧凑工具栏：新建、保存、撤销、选择、移动、缩放、识别、点/线/面、点编辑、吸附、校验、刷新。
  - 上下文栏：活动图层、编辑开关、捕捉设置、MVT 显示链路、原始 PostGIS geometry 编辑链路。
  - 左侧浏览器面板：PostgreSQL -> Local PostGIS -> public -> 表/图层树。
  - PostgreSQL 右键逻辑提示：新增、刷新、连接管理。
  - 左侧图层面板：图层勾选、激活图层、可编辑/只读状态。
  - 图层右键逻辑提示：独显、恢复可见性、缩放到图层、打开属性表。
  - 中央地图画布：地块、道路、河流、选中面、编辑顶点、吸附候选。
  - 右侧属性面板：图层与要素属性、属性表单、保存校验、属性表/字段浏览。
  - 底部状态栏：坐标、比例尺、坐标系、捕捉状态、事务保存状态、QGIS 风格提示。
- 清理了用于中间导出的临时英文副本 `设计稿/webqgis-qgis-logic-v2.svg`。

  - `Package subpath './dist/project/cua/sky_js/src/targets/windows/internal/computer_use_client_base.js' is not defined by "exports" in ... @oai/sky/package.json`
- 本轮工具列表中没有可调用的内置 `image_gen` 工具；因此无法严格完成“用 Image 插件生成设计稿”的模型生成路径。
- 当前交付的 PNG 是由确定性 SVG 导出得到，不是 imagegen 模型直接生成的图片。
- 设计稿本身暂未发现渲染 bug，PNG 已完成本地预览验证。

## 建议后续做什么

- 修复或更新 Computer Use 插件运行时后，重新打开 QGIS 做真实 UI 截图比对，补充菜单、右键菜单和停靠面板细节。
- 内置 `image_gen` 工具可用后，使用 `设计稿/WebQGIS_QGIS操作逻辑设计稿_v2_imagegen_prompt.txt` 生成一版模型高保真 UI mockup，并与当前 SVG 稿对比筛选。
- 将设计稿拆成可实施前端任务：
  - PostgreSQL 浏览器树右键新增连接弹窗。
  - 图层右键菜单与图层状态反馈。
  - 右侧要素属性/属性表联动。
  - 顶部工具栏按钮可用性与禁用态梳理。
- 后续实现时优先保持 QGIS 的密集型工作台体验，避免把 WebQGIS 改成营销页或卡片化仪表盘。
