# 图层属性表与地图显示优化进度

## 本次目标

1. 参考 QGIS 当前交互，把 WebQGIS 中无效或重复的按钮调整为可用入口。
2. 去掉图层行内“独显”按钮，保留右键菜单中的“独显图层”。
3. 缩小并统一图层右键菜单的字体、间距和尺寸，使其更接近 QGIS 桌面工具的紧凑风格。
4. 增加项目显示坐标系设置入口，先提供当前项目显示 CRS 的可见状态和切换反馈。
5. 让地图中心网格随 OpenLayers 缩放级别变化，减少静态背景感。
6. 隐藏 OpenLayers 内置缩放控件，避免和自定义工具栏重复。
7. 图层右键“打开属性表”打开可拖拽属性表，并展示要素属性记录，而不是只展示字段结构。

## 做了哪些

1. 图层面板移除了行内“独显”按钮，只保留右键菜单“独显图层”，并调整行布局。
2. 图层右键菜单改为更小的宽度、字号、行高和 padding，视觉上更接近 QGIS 菜单。
3. 工作台上下文栏新增“显示坐标系”下拉框，支持 EPSG:3857、EPSG:4326、EPSG:4490、EPSG:4547，并在状态栏展示当前显示坐标系。
4. OpenLayers 地图初始化时移除默认控件，并额外用 CSS 隐藏 `.ol-zoom` / `.ol-control`。
5. 地图网格背景移到地图容器内，根据当前 zoom 计算 CSS 变量 `--map-grid-size` 和 `--map-grid-opacity`，缩放时同步更新。
6. 后端新增 `GET /api/layers/:id/features`，从 PostGIS 读取最多 100 条属性记录，返回不含 geometry 的 Feature 摘要。
7. 前端 workspace 新增属性表状态、打开/关闭属性表动作和属性记录读取逻辑。
8. 属性表面板改为可拖拽非模态窗口，默认展示“属性数据”，同时保留“字段结构”标签页、字段过滤和排序。
9. 更新前端测试，覆盖右键独显入口、打开属性表、记录过滤、字段结构过滤和排序。
10. 验证通过：
   - `npm --workspace apps/web run typecheck`
   - `npm --workspace apps/api run typecheck`
   - `npm --workspace apps/web run test`，3 个测试文件、18 个用例通过
   - `npm --workspace apps/api run test`，3 个测试文件、8 个用例通过
   - `npm run build`

## 没做哪些

1. 没有处理 v2 设计稿相关文件；用户已说明 v2 不用管。
2. 没有恢复或处理当前工作区中无关的 `README.md` 删除、v2 设计稿删除、旧进度文档修改。
3. 显示坐标系目前是项目级 UI 状态和状态反馈，还没有实现底图/矢量数据的真实坐标转换或按 CRS 重投影显示。
4. 属性表当前只读取前 100 条属性记录，没有做分页、服务端筛选、排序或编辑单元格。
5. 属性数据默认最多展示前 12 个非几何字段，避免初版属性表过宽；完整字段仍可在字段结构页查看。
6. 没有新增 Slidev、VitePress、tsdown、Turborepo、UnoCSS 配置，本次代码路径未涉及这些能力。

## 哪里有 bug

1. 当前没有发现阻塞功能的 bug，类型检查、测试和构建均通过。
2. `npm run build` 仍有 Vite 提示：前端主 chunk 压缩后超过 500 kB。该问题不是本次改动引入的功能失败，但建议后续做代码拆分或手动 chunk。
3. 属性表缺少分页时，大表只能预览前 100 条记录，用户可能误以为已经加载全部数据。
4. 显示坐标系下拉框只改变当前项目显示设置文案，尚未影响 OpenLayers view projection 或坐标读数。

## 建议后续做什么

1. 为属性表增加分页、列显示配置、服务端过滤、排序和导出。
2. 将显示坐标系升级为完整项目 CRS 配置，联动坐标读数、比例尺、图层范围投影和必要的坐标转换。
3. 增加属性表单元格编辑和保存，和现有要素编辑写回流程统一。
4. 对 OpenLayers、Element Plus 等较大的依赖做动态导入或 `manualChunks`，消除 Vite chunk 体积警告。
5. 对 QGIS 常用菜单继续梳理，把仍不可执行的菜单项改成禁用态或补齐实际功能。
