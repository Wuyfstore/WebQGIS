# 属性表浮窗与状态栏分页修复进度

## 本次目标

- 修复属性表浮窗：打开后立即显示在地图上方，可拖动，不进入页面文档流。
- 修复 Vite/HMR 运行态不一致导致旧 CSS 残留的问题，并补属性表定位测试。
- 完善项目显示坐标系：显示 CRS 与坐标读数、状态栏、图层范围转换建立关系。
- 动态化状态栏：鼠标坐标和比例尺随 OpenLayers 地图状态更新。
- 梳理工具栏真实可用状态：区分一次性命令和持续工具。
- 补强属性表：分页、服务端筛选/排序、列宽拖拽、明确当前页/总数。
- 减少浮窗与页面布局耦合：统一属性表、连接弹窗、右键菜单、删除确认的浮层层级策略。

## 做了哪些

- `AttributeTablePanel` 使用 `Teleport to="body"`，面板为 `position: fixed`，初始显示在地图上方，并通过 `useDraggable` 支持拖拽。
- 移除属性表旧遮罩依赖，确认打开属性表前后 `body.scrollHeight` 不变化。
- 属性表新增服务端分页、搜索、排序、每页数量选择、当前页/总数提示和列宽拖拽。
- 后端 `/api/layers/:id/features` 改为返回 `{ items, total, limit, offset }`，支持 `limit/offset/search/sort/order`。
- 修复后端 `query.search` 为空时可能 500 的问题，并保证返回的 `limit/offset` 是数字。
- 状态栏改为 OpenLayers 动态输出坐标、比例尺、显示 CRS 和 zoom。
- 显示 CRS 与坐标读数联动；图层范围由后端扫描为 EPSG:4326，前端缩放到图层时转换为 EPSG:3857。
- 将“缩放/放大地图”调整为一次性“放大一级”，点击后不再残留 `zoom` 持续工具状态。
- 新增全局浮层 z-index 变量：浮动面板、菜单、模态弹窗分层管理。
- 删除确认弹窗也改为 Teleport 到 body，与连接弹窗/属性表保持一致浮层策略。
- 补充和更新 Vitest 覆盖：属性表 Teleport/fixed、分页、搜索、服务端排序、状态栏、工具栏放大命令。
- 使用真实 API 验证分页接口，使用 headless Chrome 验证属性表可见、分页、搜索、拖拽、列宽拖拽和 CRS 状态栏。

## 没做哪些

- 没有实现 EPSG:4490 / EPSG:4547 的完整 proj4 投影注册，只在状态提示中明确需要投影参数后才能精确转换。
- 没有做属性表大数据虚拟滚动，目前通过服务端分页和横向滚动缓解。
- 没有处理 Vite 大 chunk 拆包优化，本次构建仍有超过 500 kB 的提示。
- 没有处理本次任务外的脏文件：`README.md` 删除、`设计稿/*v2*` 删除、旧进度文档修改。

## 哪里有 bug

- 已修复：不带 `search` 参数直接请求属性表分页接口时，后端可能因 `query.search.trim()` 抛 500。
- 已修复：属性表如果在主布局文档流中渲染，会撑高页面或被地图/布局遮挡。
- 已修复：放大一级作为一次性命令时不应把活动工具状态残留为 `zoom`。
- 仍需关注：EPSG:4490 / EPSG:4547 目前只是提示型支持，精确转换需要后续注册投影定义。

## 建议后续做什么

- 引入 proj4 并注册 EPSG:4490 / EPSG:4547 等常用国内坐标系。
- 属性表进一步抽成独立容器组件，加入列宽持久化、字段显示开关和批量复制。
- 对 OpenLayers 相关交互增加浏览器端 E2E 测试，覆盖真实 pointer/mouse 行为。
- 对 Vite 构建做 manualChunks 或动态导入，降低主包体积。
- 对浮层增加 Escape 关闭、焦点回收和更完整的无障碍行为。
