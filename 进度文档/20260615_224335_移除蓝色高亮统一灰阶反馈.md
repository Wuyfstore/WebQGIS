# 移除蓝色高亮统一灰阶反馈

## 本次目标

- 去掉 WebQGIS 属性表 Tab、顶部按钮、菜单/表格 hover/focus 中残留的蓝色高亮。
- 保留可见的键盘焦点、选中态和 hover 反馈，但统一改为 QGIS 风格的中性灰阶。
- 保持属性表数据逻辑、顶部菜单交互逻辑、后端接口不变。

## 做了哪些

- 将全局 `.focus-ring:focus-visible` 从蓝色描边改为灰色描边，并新增 `--qgis-focus` 灰阶变量。
- 将 `--qgis-row-active` 从浅蓝改为浅灰，使菜单项、树节点、表头按钮等 hover/active 背景不再出现蓝色。
- 将顶部工具按钮 `.workbench__tool--active` 从蓝色边框/蓝色背景改为灰阶 active 状态，并保留内侧细线层次。
- 将属性表排序标记 `.attribute-table__sort-mark` 从蓝色改为 muted 灰色。
- 保留前一轮已完成的顶部菜单关闭逻辑：再次点击同菜单关闭、点击其他菜单切换、点击菜单外部关闭、按 Esc 关闭、点击菜单项后关闭。
- 保留前一轮已补充的测试：顶部菜单关闭/切换行为、属性表 Tab active class 与 `aria-selected` 切换。
- 执行并通过 `npm --workspace apps/web run test`。
- 执行并通过 `npm --workspace apps/web run typecheck`。

## 没做哪些

- 没有移除 `--qgis-blue` / `--qgis-blue-soft` 变量本身，避免影响后续可能的语义色或品牌色使用。
- 没有改绿色成功、红色危险、地图图层颜色等语义色。
- 没有改属性表数据加载、排序、分页、字段结构逻辑。
- 没有改后端接口或数据库连接逻辑。
- 没有启动浏览器做截图验证；本次通过代码检查、测试和类型检查验证。

## 哪里有 bug

- 本次修改后测试与类型检查均通过，当前未发现新增功能 bug。
- 视觉层面仍建议在浏览器里做一次人工确认，重点看属性表 Tab、顶部工具按钮 active/focus、菜单 hover/open 是否符合预期灰阶效果。
- 工作区存在本次任务外的未提交改动或删除，未纳入本次提交：`README.md` 删除、`设计稿/*v2*` 删除、`进度文档/20260612_005557_README中文化.md` 修改。
