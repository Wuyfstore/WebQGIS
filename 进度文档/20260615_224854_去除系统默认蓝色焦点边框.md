# 去除系统默认蓝色焦点边框

## 本次目标

- 处理属性表 Tab 上仍然出现的蓝色边框。
- 确保顶部按钮、属性表 Tab 等 `focus-ring` 控件不再露出浏览器或系统默认蓝色焦点样式。
- 保留可访问的键盘焦点提示，但使用项目内定义的灰阶焦点线。

## 做了哪些

- 在 `apps/web/src/styles.css` 中为 `button` 增加 `appearance: none` 和 `-webkit-appearance: none`，去掉浏览器/系统默认按钮外观。
- 在 `apps/web/src/styles.css` 中为 `.focus-ring` 增加 `outline: none`，避免默认蓝色 outline 与项目自定义 focus 样式叠加。
- 保留 `.focus-ring:focus-visible` 的灰色焦点线，键盘导航时仍有可见反馈。
- 执行并通过 `npm --workspace apps/web run test`。
- 执行并通过 `npm --workspace apps/web run typecheck`。

## 没做哪些

- 没有修改属性表 Tab 的业务逻辑、切换逻辑或 ARIA 属性。
- 没有修改顶部菜单打开/关闭交互。
- 没有修改非 `focus-ring` 控件的焦点策略。
- 没有处理工作区中原本存在的无关改动。

## 哪里有 bug

- 本次测试与类型检查均通过，未发现新增功能 bug。
- 该蓝色边框判断为浏览器/系统默认按钮 focus 外观；代码已改为由项目自定义灰阶 focus 样式接管。
- 建议在浏览器中再次确认截图位置：点击“属性数据 / 字段结构”后不应再出现蓝色边框。

## 建议后续做什么

- 对顶部工具按钮、属性表 Tab、菜单按钮做一次浏览器目视验收。
- 如果仍有个别控件出现蓝色，继续定位是否来自第三方组件默认样式或浏览器控件内部样式。
