# 修复 UnoCSS 蓝色焦点样式

## 本次目标

- 继续处理属性表 Tab 点击后仍出现蓝色边框的问题。
- 定位并移除真正生成蓝色焦点线的样式来源。
- 保持键盘焦点可见，但统一为灰阶。

## 做了哪些

- 定位到 `apps/web/uno.config.ts` 中的 `focus-ring` shortcut 仍然包含 `focus-visible:outline-blue-500`，这是截图中蓝色焦点边框的实际来源。
- 将 UnoCSS `focus-ring` shortcut 从蓝色 outline 改为灰阶 `outline-neutral-500`，并降低 outline 厚度与外扩。
- 在 `apps/web/src/styles.css` 中为 `.focus-ring` 与 `.focus-ring:focus-visible` 增加 `!important`，确保项目灰阶焦点规则优先于生成的工具类样式。
- 执行并通过 `npm --workspace apps/web run test`。
- 执行并通过 `npm --workspace apps/web run typecheck`。
- 复查 `apps/web` 下不再存在 `outline-blue`、`blue-500`、`focus-visible:outline-blue` 等蓝色焦点类。

## 没做哪些

- 没有修改属性表 Tab 的组件逻辑、数据逻辑或 ARIA 状态。
- 没有修改顶部菜单交互。
- 没有改绿色、红色等语义色。
- 没有处理工作区中已有的无关删除/修改。

## 哪里有 bug

- 本次修复前的问题是：UnoCSS 的 `focus-ring` shortcut 会生成蓝色 `outline-blue-500`，覆盖或叠加到属性表 Tab 上。
- 本次修复后测试和类型检查均通过，未发现新增功能 bug。
- 建议刷新前端页面并清理浏览器缓存后再次点击“属性数据 / 字段结构”确认蓝框消失。

## 建议后续做什么

- 若仍看到蓝色，优先重启 Vite dev server，避免 `virtual:uno.css` 热更新缓存残留。
- 后续可以把 `focus-ring` 这类基础 UI token 收敛到一个地方，避免 UnoCSS shortcut 与全局 CSS 重复定义。
