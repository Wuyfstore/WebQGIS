# 图层样式改为右键弹窗编辑

## 本次目标

- 将图层样式编辑从左侧面板的常驻展示，改为图层右键菜单中的“图层样式”入口。
- 点击“图层样式”后弹出弹窗，在弹窗中编辑填充色、边线色、透明度和线宽。
- 保留现有图层样式更新接口和地图刷新链路，不新增后端接口。

## 做了哪些

- 在 `LayerPanel.vue` 中移除常驻“图层样式”编辑面板。
- 在图层右键菜单中新增“图层样式”菜单项。
- 将 `LayerPanel.vue` 的样式编辑职责改为发出 `openStyleEditor` 事件。
- 在 `WebGisWorkbench.vue` 中新增样式弹窗状态 `styleEditorLayerId` 和 `styleEditorLayer`。
- 在 `WebGisWorkbench.vue` 中新增图层样式弹窗，包含：
  - 图层名称
  - 符号预览
  - 填充色编辑
  - 边线色编辑
  - 透明度滑块
  - 线宽滑块
  - 关闭按钮
- 弹窗编辑仍复用现有 `workspace.updateLayerStyle`，最终调用 `/api/layers/:id/style`。
- 更新 `WebGisWorkbench.test.ts`：
  - 验证右键菜单出现“图层样式”。
  - 验证常驻 `.layer-panel__style-card` 不再存在。
  - 验证点击“图层样式”后出现弹窗。
  - 验证修改填充色会调用 `apiSend("/api/layers/city/style", "PUT", { fill })`。
  - 验证关闭按钮能关闭弹窗。
- 执行并通过 `npm --workspace apps/web run test`。
- 执行并通过 `npm --workspace apps/web run typecheck`。
- 使用 Chrome 插件验收真实页面：
  - 左侧不再常驻显示“图层样式”面板。
  - 图层右键菜单包含“图层样式”。
  - 点击后弹出“图层样式”弹窗。
  - 弹窗内显示填充、边线、透明度、线宽控件。

## 没做哪些

- 没有新增或修改后端接口。
- 没有改变图层样式的数据结构。
- 没有改变地图渲染逻辑和 OpenLayers 样式应用逻辑。
- 没有处理工作区中已有的无关改动：`README.md` 删除、`设计稿/*v2*` 删除、旧进度文档修改。

## 哪里有 bug

- 本次测试、类型检查和 Chrome 验收均通过，未发现新增功能 bug。
- 样式编辑仍为即时保存模式，用户拖动或 change 控件后直接调用接口；本次没有增加“应用/取消”草稿机制。
- 弹窗关闭不会回滚已经保存的样式，这是沿用原有即时编辑行为。

## 建议后续做什么

- 后续可以把样式弹窗拆成独立 `LayerStyleDialog.vue`，减少 `WebGisWorkbench.vue` 模板体积。
- 若需要更接近 QGIS，可增加“应用 / 确定 / 取消”三按钮和本地草稿，避免每次控件变化都请求接口。
- 可以补一个浏览器级视觉回归截图，覆盖图层右键菜单和样式弹窗。
