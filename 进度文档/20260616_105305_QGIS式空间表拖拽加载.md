# QGIS式空间表拖拽加载

## 本次目标

- 将数据库浏览器中的空间表目录与当前地图已加载图层拆开。
- `Docker PostGIS Test` 展开后显示可用空间表，空间表可双击添加，也可拖到地图或图层面板添加。
- 只有添加后的空间表才进入“图层”面板、挂载到 OL 地图、成为活动图层。
- 浏览器硬刷新后不持久化已加载图层；同一页面内刷新工作区只保留仍存在的已加载图层。

## 做了哪些

- 在工作区状态中新增 `availableLayers` 作为空间表目录，新增会话级 `loadedLayerIds` 表示已加载图层。
- 将 `layers` 改为从 `availableLayers + loadedLayerIds` 派生，避免刷新或扫描后自动把全部空间表加入图层面板。
- 调整 `refreshAll()`：只刷新数据源和空间表目录，并协调仍存在的已加载图层。
- 调整 `scanDatasource()`：扫描后只更新空间表目录，状态提示“可拖入地图添加图层”。
- 新增 `loadLayer(layerId)`：校验空间表存在，加入已加载集合和可见集合，设为活动图层，清空编辑草稿。
- `DatasourcePanel` 在 PostgreSQL 连接下展示空间表节点，显示 `schema.table`、几何类型、SRID、可编辑/只读/已加载状态。
- `DatasourcePanel` 支持空间表节点双击加载和 `draggable=true`，拖拽 payload 使用 layer id。
- `MapCanvas` 增加空间表 drop 区和灰阶拖放反馈。
- `LayerPanel` 空状态改为“从浏览器拖入空间表以添加图层”，并支持作为 drop 区接收空间表。
- `WebGisWorkbench` 统一处理双击/拖放加载，并在 `nextTick` 后调用 `editor.zoomToLayerExtent(layerId)`。
- 更新单元测试覆盖目录与图层分离、双击加载、拖到地图、拖到图层面板、重复添加不重复、既有编辑流程先加载图层。

## 没做哪些

- 未新增或修改后端接口，继续使用现有 `/api/datasources/:id/scan`、`/api/layers`、MVT、features、style 接口。
- 未实现工程保存、图层移除持久化、图层排序持久化。
- 未改变空间表默认样式生成逻辑。
- 未改变属性表数据逻辑、编辑保存接口或后端数据库结构。

## 哪里有 bug

- 未发现本次功能性 bug。
- Chrome 自动化的物理鼠标拖拽没有成功携带 HTML5 `DataTransfer`，因此浏览器实测里使用双击完成真实页面加载验证；拖放路径已在 Vitest 中用带 payload 的原生事件覆盖。页面节点已确认 `draggable=true`，地图和图层面板均有 drop 处理。

## 建议后续做什么

- 增加真实浏览器 E2E 用例，使用 Playwright 的 HTML5 drag/drop 方式验证拖放链路。
- 给空间表节点增加更多 QGIS 风格目录层级，例如按 schema 分组。
- 增加图层移除、图层排序、工程保存与恢复。
- 如需更清晰的拖放可见反馈，可给拖拽源和目标加轻量状态提示，但保持灰阶风格。
