# QGIS式图层编辑流程修复

## 本次目标

- 修复空间表无法拖动到“图层”面板添加图层的问题。
- 按 QGIS 式工作流完善图层右键菜单：支持移除图层、开启编辑、编辑中关闭编辑。
- 编辑默认关闭；关闭时不显示活动图层编辑相关面板，开启编辑后才显示。
- 保持图层加载、编辑、移除状态一致，避免重复加载或残留编辑状态。

## 做了哪些

- 新增 `layerDrag` 工具，统一空间表拖拽 payload 写入、检测和读取逻辑。
- 在数据源浏览器拖拽空间表时写入自定义 MIME 与 `text/plain`，并向父组件记录当前拖拽的 layer id。
- 在地图画布和图层面板 drop 区增加 fallback：当真实浏览器在 drop 阶段读不到 `DataTransfer.getData()` 时，仍可使用父组件保存的拖拽 layer id 完成加载。
- 在 `LayerPanel` 右键菜单增加：
  - `开启编辑`
  - `关闭编辑`
  - `移除图层`
- 在 `useWebGisWorkspace` 新增 `removeLayer(layerId)`，移除已加载图层时同步清理可见图层、上一次可见性、属性表、活动图层和草稿状态。
- 在 `WebGisWorkbench` 调整编辑流程：
  - 默认编辑关闭。
  - 支持从顶部工具栏、地图编辑入口、图层右键菜单开启/关闭编辑。
  - 切换到非编辑中图层时自动关闭编辑并清空草稿。
  - 移除正在编辑的图层时同步关闭编辑。
- 将右侧 `EditInspector` 改为仅在当前活动图层处于编辑中时渲染。
- 将地图内编辑浮层改为仅在编辑中显示，关闭编辑时不再显示活动图层编辑浮层。
- 更新测试，覆盖图层面板拖拽 fallback、右键开启/关闭编辑、移除图层、编辑面板显隐。

## 没做哪些

- 未修改后端接口、PostGIS 扫描接口、MVT 接口或要素保存接口。
- 未实现工程文件保存、图层顺序持久化、图层移除持久化。
- 未处理工作区中既有的无关改动：`README.md` 删除、`设计稿/*v2*` 删除、`进度文档/20260612_005557_README中文化.md` 修改。
- 未使用真实浏览器自动化完成最终拖放验证，因为当前可用工具未暴露 Browser/Chrome 控制接口，且项目未安装 Playwright。

## 哪里有 bug

- 本次修复前的主要问题：
  - 空间表拖到图层面板时，真实浏览器可能在 drop 阶段无法读取 `DataTransfer.getData()`，导致添加图层无效。
  - 图层右键菜单缺少移除图层、开启编辑、关闭编辑入口。
  - 编辑关闭时右侧活动图层编辑面板和地图编辑浮层仍显示，和 QGIS 式编辑流程不一致。
- 本次修复后，组件测试与类型检查未发现新的功能性 bug。
- 剩余风险：
  - 尚未做真实浏览器端到端拖放测试。
  - 旧的无关工作区改动仍需后续单独处理，避免长期污染 `git status`。

## 建议后续做什么

- 增加 Playwright 或等价 E2E 测试，覆盖真实浏览器拖拽空间表到地图、拖到图层面板、双击加载三条链路。
- 给图层右键菜单增加更多 QGIS 常用动作，例如重命名图层、复制图层、打开图层属性。
- 增加“关闭编辑时存在未保存草稿”的确认弹窗，避免误关编辑丢失草稿。
- 单独整理当前工作区中遗留的 README 删除、设计稿 v2 删除和旧进度文档修改。
