# 地图右键与属性表增强

## 本次目标

- 地图画布右键支持复制坐标、复制比例尺、复制 Zoom。
- 移除顶部工具栏里重复的“刷新图层”宽按钮，保留左侧已有刷新入口。
- 图层右键菜单增加“刷新图层”。
- 将底图/项目坐标系选择改成更接近 QGIS 状态栏 CRS 控件的排版。
- 属性表新增属性计算器，并给开发人员提供安全 SQL 查询入口。
- 排查地图网格来源；若不是 OpenLayers 自带网格，则改为 OpenLayers 图层实现。
- 完成后生成进度文档并提交、推送到 `main`。

## 做了哪些

- 在 `MapCanvas` 中新增地图右键菜单：
  - `复制坐标`
  - `复制比例尺`
  - `复制 Zoom`
- 复制动作使用浏览器剪贴板，并通过工作台状态栏提示已复制内容。
- 移除工具栏最右侧重复的“刷新图层”宽按钮，保留原有图标刷新按钮。
- 在 `LayerPanel` 图层右键菜单中新增 `刷新图层`，调用 OpenLayers 图层 source refresh。
- 将工作台坐标系选择改为 `项目 CRS` 灰阶面板，包含坐标系下拉框和当前投影说明，更接近 QGIS 状态栏/项目 CRS 控件的布局。
- 排查并确认原地图网格是 CSS `linear-gradient` 背景，不是 OpenLayers 自带网格。
- 将 CSS 网格替换为 OpenLayers `Graticule` 图层，保留灰阶、虚线、标签显示。
- 属性表新增两个 Tab：
  - `属性计算器`
  - `SQL 查询`
- 属性计算器支持选择可编辑字段、填写表达式和可选 WHERE 条件。
- SQL 查询支持 `{layer}` 占位符表示当前图层表，并展示查询结果表格。
- 后端新增接口：
  - `POST /api/layers/:id/query`
  - `POST /api/layers/:id/calculate`
- 后端 SQL 安全控制：
  - SQL 查询仅允许单条 `SELECT`。
  - 禁止 DDL/DML/危险函数/注释/多语句。
  - SQL 必须查询当前图层表或使用 `{layer}`。
  - 服务端外层强制 `LIMIT`，最大 200。
  - 查询在显式 `BEGIN READ ONLY` 事务中执行。
  - 属性计算表达式只允许当前图层字段、少量安全函数、CASE/条件关键字和基础运算符。
  - 属性计算目标字段必须是当前图层可编辑字段。
- 补充前后端测试，覆盖地图右键复制、图层右键刷新、属性计算器、SQL 查询和 SQL 注入防护。

## 没做哪些

- 未使用真实 QGIS 插件截图对照，因为本轮主要按 QGIS 风格目标直接落实现；UI 已按 QGIS 状态栏 CRS 控件的灰阶紧凑布局调整。
- 未实现完整 QGIS 表达式引擎，只实现服务端安全白名单表达式。
- 未实现任意跨表 SQL、JOIN、写 SQL 或数据库管理功能，避免扩大注入与误操作风险。
- 未新增 Playwright 真实浏览器 E2E。
- 未处理工作区既有无关改动：`README.md` 删除、`设计稿/*v2*` 删除、`进度文档/20260612_005557_README中文化.md` 修改。

## 哪里有 bug

- 本次修复前的问题：
  - 地图网格由 CSS 背景绘制，并非 OpenLayers 网格图层。
  - 地图画布右键没有复制坐标/比例尺/Zoom。
  - 图层右键缺少刷新图层。
  - 顶部存在重复刷新入口。
  - 属性表缺少属性计算器和开发用 SQL 查询入口。
- 本次修复后，单元测试、类型检查和构建均通过。
- 剩余风险：
  - SQL 查询虽然已做严格限制，但仍建议生产环境增加权限隔离、审计日志和查询超时。
  - 属性计算表达式是保守白名单，不等同于完整 QGIS 表达式语法。
  - Vite 构建提示前端 chunk 超过 500 kB，这是体积优化提示，不影响构建成功。

## 建议后续做什么

- 增加 Playwright E2E，覆盖真实浏览器右键菜单、剪贴板、属性计算和 SQL 查询。
- 为 SQL 查询和属性计算增加后端审计日志、超时控制和角色权限。
- 按 QGIS 表达式语法继续扩展属性计算器，支持函数帮助、字段插入、表达式预览。
- 对 OpenLayers Graticule 标签样式继续微调，使其更接近 QGIS 网格/经纬网效果。
- 单独整理工作区遗留的 README 删除、设计稿 v2 删除和旧进度文档修改。
