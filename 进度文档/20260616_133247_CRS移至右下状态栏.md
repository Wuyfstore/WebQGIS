# CRS 移至右下状态栏进度文档

## 本次目标

- 按 QGIS 风格将 CRS 选择控件放到 WebQGIS 底部右下角状态栏。
- 移除底部状态栏中原有的长文本 `EPSG:3857 显示 / EPSG:4326 数据源 · Web Mercator 显示坐标` 一类说明。
- 保持 CRS 切换功能可用，不影响空间表加载、图层、编辑、属性表等既有流程。
- 完成验证后提交并推送到 `main` 分支。

## 做了哪些

- 从 `WebGisWorkbench.vue` 的编辑上下文栏中移除了原来的大块 CRS 面板。
- 从底部状态栏中移除了直接显示的 `projectionLabel` 长文本。
- 在底部状态栏右侧新增紧凑 CRS 下拉控件，使用 `CRS + select` 的 QGIS 式状态栏布局。
- 保留 `workspace.setDisplayProjection(...)` 绑定，切换 CRS 时仍会更新项目显示坐标系。
- 移除了旧 `.workbench__crs-panel` / `.workbench__crs-title` / `.workbench__crs-hint` 样式。
- 新增 `.workbench__status-crs`、`.workbench__status-crs-label`、`.workbench__status-crs-select` 样式，控制右下角 CRS 控件尺寸和灰阶外观。
- 调整状态文字溢出处理，避免状态信息挤压右侧 CRS 控件。
- 更新 `WebGisWorkbench.test.ts`：
  - 断言底部状态栏不再渲染长 CRS 说明文本。
  - 断言 CRS 下拉位于 `.workbench__statusbar`。
  - 断言编辑上下文栏不再包含 CRS 下拉。
  - 断言默认项目 CRS 下拉值为 `EPSG:3857`。

## 没做哪些

- 未修改后端接口。
- 未修改空间表扫描、拖拽加载、图层渲染、属性表、编辑保存等业务逻辑。
- 未移除 `useOpenLayersEditor` 内部的 `projectionLabel` 计算，因为地图复制/其他状态能力后续仍可能复用。
- 未处理工作区中已有的无关未提交变更，包括 README 删除、设计稿删除和既有进度文档修改。

## 哪里有 bug

- 本次实现后第一次执行 `npm --workspace apps/web run test` 时，新增断言误认为 CRS 默认值应为 `EPSG:4326`，实际工作区默认显示 CRS 是 `EPSG:3857`，导致 `WebGisWorkbench.test.ts` 中 1 个测试失败。
- 已修正测试断言为 `EPSG:3857`，之后完整测试通过。
- 当前未发现本次改动引入的功能 bug。
- `git diff --check` 仅提示部分文件下次 Git 触碰时 LF 会替换为 CRLF，没有发现空白错误。

## 建议后续做什么

- 用浏览器打开 WebQGIS 做一次人工视觉确认，重点检查右下角 CRS 下拉与状态文本在不同窗口宽度下是否拥挤。
- 后续如果继续贴近 QGIS，可以把 CRS 下拉改成点击状态栏 CRS 文本后弹出更完整的 CRS 选择对话框。
- 对底部状态栏进行一次整体信息分组梳理，避免坐标、比例尺、缩放、捕捉、状态提示在窄屏下过密。
