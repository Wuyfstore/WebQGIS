# QGIS 式选择、属性计算器与坐标设置进度文档

## 本次目标

- 选择要素工具点击后提供模式选项：点击选择、范围选择、自定义范围选择。
- 使用 Computer Use 查看 QGIS 中选择工具、字段计算器入口和 CRS/坐标设置的设计，并结合 WebQGIS 当前 UI 调整实现。
- 优化属性表中的属性计算器布局，使其更接近 QGIS 的字段/函数/表达式编辑工作流。
- 拓展 OL 坐标显示设置，在右下角 CRS 区域提供 QGIS 式 CRS/坐标设置窗口。
- 完成后执行测试、生成进度文档、提交并推送到 `main` 分支。

## 做了哪些

- 使用 Computer Use 打开 QGIS Desktop 3.44.10，并观察到：
  - QGIS 主界面有独立“选择工具栏”，包含“选择要素”“按值选择要素”“取消选择”“按位置选择”等入口。
  - QGIS 右下角状态栏包含坐标、比例尺、放大镜、旋转、渲染和 `EPSG:4326` 按钮。
  - 点击右下角 EPSG 按钮会打开“工程属性 - CRS”，包含无 CRS、过滤、最近使用 CRS、预定义 CRS、CRS 详情和应用/确定操作。
  - 未加载可编辑图层时，QGIS 的字段计算器按钮处于禁用状态。
- 在 `useOpenLayersEditor.ts` 中新增选择模式状态：
  - `click`：点击选择。
  - `extent`：范围选择。
  - `customExtent`：自定义范围选择。
  - 新增 `activateSelectionMode()`，选择不同模式时更新工具状态和状态栏提示。
- 在工作台工具栏中将“选择”按钮改为 QGIS 式分裂按钮：
  - 主按钮执行当前选择模式。
  - 右侧小按钮展开三种选择模式。
  - 支持再次点击/点外部/Esc 后关闭菜单。
- 在 `useOpenLayersEditor.ts` 中拓展坐标格式化：
  - 支持坐标小数位数。
  - 支持 `X,Y` 与 `Y,X` 坐标顺序。
- 在右下角 CRS 状态区新增“设置”按钮，并实现 QGIS 式 CRS 设置弹窗：
  - 左侧分类栏：CRS、变换、坐标显示。
  - CRS 区域：无 CRS、过滤、最近使用坐标参照系、预定义坐标参照系、坐标系详情。
  - 坐标显示区域：显示 CRS、坐标顺序、小数位数。
  - 确定/应用后更新状态提示。
- 重构属性表的属性计算器：
  - 左侧字段和值/函数列表，点击字段或函数可插入表达式。
  - 中间表达式编辑区、常用操作符、SQL 预览。
  - 右侧输出字段、仅更新匹配要素、目标图层摘要和执行按钮。
  - 保持原有 `calculate` 事件与后端接口不变。
- 更新测试：
  - 补齐编辑器 mock 的 `selectionMode` 和 `activateSelectionMode`。
  - 验证选择工具下拉中有范围选择，并能调用选择模式切换。
  - 验证右下角 CRS 设置弹窗的 QGIS 式结构和坐标顺序设置。
  - 验证新版属性计算器仍能提交计算请求。

## 没做哪些

- 未新增后端接口。
- 未实现真正的空间范围查询/框选结果批量选择；本次只完成选择模式入口、状态切换和提示。
- 未实现 QGIS 完整字段计算器函数全集，只加入字符串、数学、条件类常用函数入口。
- 未注册 EPSG:4490、EPSG:4547 的完整 proj4 参数；保留原有显示转换能力和提示。
- 未处理工作区中已有的无关未提交改动。

## 哪里有 bug

- 当前未发现本次改动引入的已知 bug。
- 开发过程中出现过一次测试失败：`WebGisWorkbench.test.ts` 中旧 mock 缺少 `selectionMode`，导致读取 `selectionMode.value` 报错；已补齐 mock。
- 开发过程中出现过一次属性计算器测试断言失败：jsdom 的 `textContent` 不包含 textarea 的 `value`；已改为读取预览 textarea 的 `value`。

## 建议后续做什么

- 后续接入真正的 OL 框选交互：拖拽生成 extent，并按空间范围查询后端要素。
- 为“自定义范围选择”增加 xmin/ymin/xmax/ymax 输入面板，并接入后端空间过滤接口。
- 将属性计算器函数列表改为后端能力驱动，避免 UI 展示服务端不支持的函数。
- 为 CRS 设置增加 CRS 搜索过滤、最近使用 CRS 持久化、投影参数注册和坐标变换配置。
- 用浏览器人工检查工具栏分裂按钮、属性计算器三栏和 CRS 弹窗在不同窗口尺寸下的视觉密度。
