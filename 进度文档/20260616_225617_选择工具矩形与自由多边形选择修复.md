# 选择工具矩形与自由多边形选择修复

## 本次目标

- 修复“选择”工具中的范围选择（矩形）无法使用的问题。
- 修复“选择”工具中的自定义范围选择（自由绘制多边形）无法使用的问题。
- 调整“选择”按钮交互：工具栏只显示图标，右键打开选择方式；菜单中显示图标和标题，不再用省略号表示多选项。
- 任务完成后生成进度文档，并提交推送到 main 分支。

## 做了哪些

- 在 OpenLayers 编辑器 composable 中接入 `DragBox`，范围选择模式会在地图上拖拽矩形并从当前已加载矢量瓦片候选要素中选取命中要素。
- 将自定义范围选择从原来的“输入范围坐标”语义改为“自由绘制多边形范围”，使用 `Draw` 的 Polygon 交互，双击结束后执行范围命中并回源读取要素。
- 增加选择草图图层和矩形框灰阶样式，让矩形/多边形选择反馈可见且与当前 QGIS 灰阶 UI 风格一致。
- 调整工具栏“选择”按钮：主按钮只保留图标，左键启用当前选择方式，右键打开选择方式菜单。
- 移除选择工具的省略号拆分按钮，菜单项改为图标 + 标题，不再显示冗长说明文本。
- 补充测试覆盖：选择按钮右键菜单、菜单图标、矩形选择模式、自定义自由多边形选择模式、选择模式状态文案、候选要素范围命中判断。

## 没做哪些

- 没有实现多要素批量选择列表；当前仍按现有工作区模型回源读取第一个命中的要素。
- 没有新增后端空间查询接口；本次基于前端当前已加载的 MVT 候选要素执行选择。
- 没有修改图层编辑、属性表、样式编辑、PostGIS 数据源扫描等其它流程。
- 没有处理工作区中已有的无关删除和旧文档修改。

## 哪里有 bug

- 本次修复前，矩形范围选择和自定义范围选择只有状态提示，没有真实地图交互，因此点击后看起来无法使用。
- 当前已知限制：范围选择基于当前屏幕/瓦片已加载的候选要素，若目标要素所在瓦片尚未加载，可能需要先缩放或平移到目标区域后再选择。
- 当前未发现新增阻塞 bug。

## 建议后续做什么

- 后续可新增后端范围查询接口，按矩形或多边形直接在 PostGIS 中执行 `ST_Intersects`，从而支持跨当前瓦片的可靠范围选择。
- 可扩展为多要素选择集合，并在属性表或右侧面板展示选择集。
- 可给状态栏增加当前选择模式提示，减少用户不知道“右键切换选择方式”的情况。
