# CRS 数据库查询与地方坐标系 CRUD 进度文档

## 需求

1. 右下角 CRS 改成设置按钮样式，不再用 select。
2. 右下角不再单独显示“设置”按钮，直接点击坐标系打开设置。
3. 使用数据库坐标系表，增加坐标系数量，并支持模糊查询。
4. 设置里允许添加地方坐标系，并提供 CRUD 接口。

## 本次实现

- 后端新增 `CrsModule`：
  - `GET /api/crs/search?q=&limit=`：从已保存 PostGIS 数据源的 `public.spatial_ref_sys` 查询坐标系。
  - 支持按 `srid`、`auth_name`、`auth_srid`、`srtext`、`proj4text` 模糊查询。
  - 查询失败的数据源会跳过，保留自定义坐标系和常用 EPSG 兜底项。
- 后端新增地方坐标系 CRUD：
  - `GET /api/crs/custom`
  - `POST /api/crs/custom`
  - `PUT /api/crs/custom/:id`
  - `DELETE /api/crs/custom/:id`
  - 自定义地方坐标系存储在项目 JSON 数据文件 `custom-crs.json` 中。
- 前端工作台改造：
  - 右下角 CRS 状态区从下拉框改为 `CRS + 当前坐标系按钮`。
  - 移除右下角单独的“设置”按钮，点击当前 CRS 即可打开设置弹窗。
  - CRS 弹窗中的过滤框改为真实搜索输入，联动 `/api/crs/search`。
  - 坐标系列表显示来源：数据库、地方、内置。
  - 弹窗中新增“地方坐标系”区域，支持新增、编辑、删除和选择自定义 CRS。
  - 未注册到 OpenLayers 的 CRS 坐标读数安全降级为空坐标，避免选择数据库/地方 CRS 后鼠标移动触发投影转换异常。
- 类型与测试：
  - 前端新增 `CrsDefinition`、`CustomCrsPayload` 类型。
  - 工作台测试增加 CRS API mock，并验证状态栏 CRS 按钮、数据库搜索入口和地方坐标系区域。

## 说明

- 本轮实现使用 PostGIS 标准坐标系表 `public.spatial_ref_sys` 作为数据库坐标系来源。
- 自定义地方坐标系目前完成元数据和 Proj4 参数 CRUD；精确投影转换仍需后续接入 proj4 注册流程。
