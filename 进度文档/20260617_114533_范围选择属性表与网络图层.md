# 范围选择属性表与网络图层进度

## 本次目标

- 修复范围选择、自定义范围选择只返回一个数据的问题。
- 范围选择与自定义范围选择仅允许左键绘制，右键不触发绘制。
- 优化坐标系选择和地方坐标系定义 UI，参考 QGIS 工程属性布局，避免遮挡与大面积空白。
- 属性表支持下载当前表格数据。
- 字段计算器与 SQL 查询改为无遮罩、可拖动的独立弹窗；SQL 查询结果进入属性表主 table。
- 优化字段计算器布局。
- 参考 QGIS Browser 的 XYZ Tiles、WMS/WMTS 连接和加载逻辑，在当前项目中复现基础浏览与加载能力。
- 使用多 agent 协同审查并汇总处理。

## 做了哪些

- 范围选择不再只取第一个命中要素，改为收集选择几何内所有可查询要素，按图层分组并去重后打开属性表。
- 后端属性分页接口新增 `ids` 查询参数，支持范围选择按多个主键回源查询。
- 矩形范围选择 `DragBox` 和自定义范围 `Draw` 均加入左键/无修饰键条件，右键不会开始选择绘制。
- 属性表主 table 统一渲染普通属性数据或 SQL 查询结果；SQL 查询后自动回到“属性数据”视图显示结果。
- 属性表新增 CSV 下载，下载内容取当前主 table 的列和行，普通记录与 SQL 结果都可导出。
- 字段计算器与 SQL 查询从 tab 改为 `Teleport` 到 body 的独立可拖动弹窗，无遮罩，并保留关闭按钮。
- 字段计算器改为 QGIS 式三栏布局：字段/函数、表达式与预览、输出与过滤。
- CRS 设置弹窗改为左侧分类、右侧搜索、CRS 列表、详情、坐标显示、地方坐标系的紧凑布局，并为列表/地方坐标系设置滚动边界。
- 浏览器面板新增 `XYZ Tiles` 与 `WMS/WMTS` 根节点，默认提供 OpenStreetMap XYZ，支持手工新增 XYZ/WMS/WMTS 连接。
- 工作区新增 Web 栅格图层模型，PostGIS 刷新不会覆盖 Web 图层。
- OpenLayers 挂载支持 PostGIS MVT、XYZ、WMS、WMTS 基础栅格源；范围选择只遍历矢量瓦片图层。
- Web 栅格图层禁用属性表、编辑、字段计算器、SQL 查询等 PostGIS 专属能力，仅保留加载、显示/隐藏、刷新、移除等基础操作。
- 使用两个并行 agent 做只读审查：一个审查属性表半重构断点，一个审查 XYZ/WMS/WMTS 低冲突接入方案。
- 执行并通过：
  - `npm --workspace apps/web test -- src/components/workbench/WebGisWorkbench.test.ts`
  - `npm --workspace apps/web test -- src/composables/useOpenLayersEditor.test.ts`
  - `npm --workspace apps/api run test`
  - `npm --workspace apps/api run typecheck`
  - `npm run typecheck`

## 没做哪些

- 未实现 WMS/WMTS GetCapabilities 自动解析与子图层树自动发现；当前为手工填写连接参数的基础加载。
- 未做浏览器真实截图验收；本次以组件测试、类型检查和 API 测试作为验证依据。
- 未处理工作区中任务开始前已存在的无关删除/修改，包括 `README.md`、`设计稿/*` 和旧进度文档修改。
- Computer Use 插件此前连接 Windows helper 失败，未继续使用 Windows 桌面自动化；本次采用代码与测试验证。

## 哪里有 bug

- 未发现本次改动引入的已知阻塞 bug。
- WMTS 当前依赖手工 `url/layer/matrixSet/style/format`，不同服务的 matrix id 命名可能不完全一致，复杂 WMTS 仍需要 GetCapabilities 解析。
- Web 栅格图层暂不支持本地样式编辑，仅沿用默认透明度和源参数。

## 建议后续做什么

- 增加 WMS/WMTS GetCapabilities 解析，自动列出服务子图层、样式、格式、TileMatrixSet 与范围。
- 给 Web 栅格图层增加本地透明度和重命名设置弹窗。
- 为范围选择增加更贴近真实 OpenLayers 交互的集成测试，验证多个瓦片要素命中后请求 `ids` 参数。
- 增加 CSV 下载内容的独立单元测试，覆盖逗号、换行、引号和 SQL 结果导出。
- 用浏览器或 Playwright 做一次 CRS 弹窗和属性工具窗的视觉回归截图。
