# 本次目标

1. 范围选择 / 自定义范围选择命中后，像点选一样在地图上高亮要素，不再自动打开属性表。
2. 尝试使用 Computer Use 打开 QGIS 并观察 PG 图层操作逻辑，尽量向 QGIS 的“画布选择高亮、属性表另行打开”的流程靠齐。
3. 对当前功能流程和 QGIS 操作习惯做对齐：地图选择优先反馈到画布，属性表保持手动入口。
4. 针对 90w+ 点线大表继续做响应优化。
5. 使用多 agent 协作完成只读核查与主线实现。

# 做了哪些

1. 使用 `vue`、`vue-best-practices`、`vue-router-best-practices`、`vue-testing-best-practices`、`web-design-guidelines`、`nestjs-best-practices`、`chrome:control-chrome`、`computer-use:computer-use` 相关技能要求完成本轮处理。
2. 使用多 agent 只读核查：
   - 交互核查确认点选走 `readFeature -> loadEditableFeature` 高亮，范围选择此前写死为 `openAttributeTableForFeatureIds`。
   - 性能核查确认大表慢点主要集中在 MVT 候选数量、瓦片属性负载、ids 过滤额外 count、搜索和深分页等。
3. 后端 `selectFeatures` 现在返回 `ids` 之外，还返回最多 500 条用于地图高亮的 GeoJSON `features`，前端无需再为了范围选择打开属性表。
4. 前端 `useOpenLayersEditor` 新增只读选择高亮图层：
   - 点选仍使用原来的可编辑草稿高亮。
   - 范围选择 / 自定义范围选择使用独立 `selectedFeatureSource` 高亮，不污染单要素编辑状态。
   - 范围内无结果时清空范围高亮。
5. 移除工作台向 editor 传入的范围选择打开属性表回调，属性表入口仍保留给图层面板 / 手动操作。
6. 大表性能优化：
   - MVT 单瓦片候选按缩放级别限制：低缩放 5000，中等缩放 12000，高缩放 25000。
   - MVT 属性列从最多 24 个收窄到最多 4 个非主键、非 geometry 名称字段，降低瓦片体积。
   - 属性表 ids 过滤且无搜索时不再执行额外 `count(*)`，直接用 ids 数量作为总数。
7. Chrome 本地烟测：
   - `http://127.0.0.1:5173` 可加载 WebQGIS 主界面。
   - 页面无 console error。
8. 验证命令：
   - `npm run typecheck` 通过。
   - `npm --workspace apps/api run test` 通过，18 个测试通过。
   - `npm --workspace apps/web run test` 通过，39 个测试通过。
   - `GET http://127.0.0.1:4100/api/health` 返回正常。

# 没做哪些

1. 没有实际完成 QGIS 内 PG 连接和操作录屏式观察，因为 Computer Use 插件运行时在列出 Windows 应用前失败。
2. 没有改数据库结构，例如给两张大表补真实主键、唯一约束、SRID 约束或空间索引。
3. 没有实现 QGIS 式“只显示选中要素 / 当前地图范围要素 / 全部要素”的属性表筛选 UI。
4. 没有优化全文搜索和深分页；当前 `to_jsonb(t)::text ilike` 在 90w+ 表上仍可能慢。
5. 没有继续处理顶部第三行 UI 历史需求，本轮按当前 active goal 聚焦范围选择、QGIS 对齐和性能。

# 建议后续做什么

1. 修复或更新 Computer Use 插件运行时后，重新打开 QGIS，对照真实 PG 图层的选择、属性表、当前范围过滤流程再做一轮交互细化。
2. 数据库层给两张 90w+ 点线表补真实主键 / 唯一约束和有效 SRID 约束，并确认 geometry 上存在 GiST/SP-GiST 空间索引。
3. 属性表增加 QGIS 风格过滤模式：全部要素、选中要素、当前地图范围要素。
4. 大表搜索改成字段级搜索、pg_trgm 或全文索引，不再默认对整行 JSON 文本 `ilike`。
5. 深分页改 keyset pagination，避免大 offset 翻页越来越慢。
6. 对 MVT 候选上限做 UI 提示或按图层配置化，必要时支持按比例尺隐藏 / 简化 / 聚合。
