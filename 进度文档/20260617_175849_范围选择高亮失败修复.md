# 范围选择高亮失败修复

## 本次目标

- 修复服务端范围选择返回 ids 后，前端选择结果高亮失败的问题。
- 前端只处理 geometry 非空且结构合法的 GeoJSON Feature，遇到 `geometry: null` 或异常 feature 时跳过，不允许中断范围选择流程。
- 修复已渲染瓦片 fallback 高亮时把 OpenLayers RenderFeature 直接塞进普通 Feature 导致的报错。
- 修复选择方式右键菜单 `selectionToolRef.value?.contains is not a function` 及菜单项点击失效问题。
- 使用 Chrome 实测“范围选择”和“自定义范围选择”，确认返回后有橙色高亮、不打开属性表、Console 无相关错误。

## 做了哪些

- 后端 `/api/layers/:id/features/select` 查询结果从只返回 `id` 改为返回 `id, feature`，让前端有可用于高亮的 GeoJSON Feature。
- 前端新增 GeoJSON Feature 合法性判断，过滤空 geometry、缺少 geometry type 的异常 feature。
- `loadSelectedFeatures()` 增加 `try/catch` 和 `parsed.getGeometry()` 判断，异常 feature 跳过不中断。
- `loadRenderedSelectionFeatures()` 改为安全转换已渲染瓦片 geometry；RenderFeature 使用 OpenLayers `toFeature()` 转为普通 Feature，普通 Geometry 使用 GeoJSON roundtrip 克隆，无法转换时跳过并 warning。
- 选择工具右键菜单 ref 改为函数 ref，保存真实 HTMLElement，避免 v-for 模板 ref 非 DOM 形态导致 contains 报错，也避免菜单项点击前被外部 pointerdown 提前关闭。
- 补充服务端 SQL 断言、GeoJSON 合法性测试、RenderFeature fallback 转换测试。
- 使用 Chrome 实测：
  - 范围选择：状态显示 `范围选择完成：高亮 10 个要素`，属性表未打开，Console error 为空。
  - 自定义范围选择：状态显示 `范围选择完成：高亮 5 个要素`，属性表未打开，Console error 为空。

## 没做哪些

- 没有修改无关的 README、设计稿删除状态、旧进度文档改动。
- 没有调整图层坐标系配置；当前实测 `public.china_2025_province` 在浏览器树中显示为 SRID 4326，项目显示坐标系仍为 EPSG:3857。
- 没有新增完整端到端自动化测试，已通过 Chrome 人工自动化验收覆盖本次路径。

## 哪里有 bug

- 本次发现并修复：右键选择菜单因为 ref 绑定在 v-for 内，点击菜单项时外部 pointerdown 会先关闭菜单，导致选择模式没有切换。
- 本次发现并修复：服务端选择 SQL 构造了 GeoJSON feature，但最终 select 只返回 id，前端无法稳定高亮。
- 本次发现并修复：RenderFeature 不是普通 Observable Geometry，直接放进 `new Feature({ geometry })` 会触发 `target.addEventListener is not a function`。
- 当前未发现新的阻塞 bug。

## 建议后续做什么

- 增加 Playwright E2E 测试，自动覆盖范围选择、自定义范围选择、属性表不弹出、Console 无错误。
- 对选择结果增加更明确的前端 warning 统计，例如跳过了多少个非法 feature。
- 继续核对测试数据表的真实 SRID，避免口头坐标系和浏览器扫描结果不一致。
