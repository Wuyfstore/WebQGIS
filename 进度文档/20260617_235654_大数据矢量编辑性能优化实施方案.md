# 大数据矢量编辑性能优化实施方案

## 本次目标

- 将“大数据表既要显示快，又要支持 WebQGIS 编辑后快速响应”的优化思路整理成详细设计文档。
- 拆分为 Codex agent 可执行的阶段目标、改动范围和验收标准。

## 做了哪些

- 核对当前项目中浏览、选择、编辑、保存、瓦片生成相关代码入口。
- 新增设计文档 `设计文档/大数据矢量编辑性能优化实施方案.md`。
- 文档中明确总体架构：
  - 浏览层：MVT/缓存 MVT/PMTiles 等负责大数据快速浏览。
  - 选择高亮层：少量 GeoJSON 负责选择结果高亮。
  - 编辑覆盖层：只加载当前编辑要素，保证编辑精确和即时响应。
  - 增量刷新层：保存后先更新覆盖层，再刷新受影响瓦片。
- 将实施拆分为 7 个阶段：
  - 阶段 0：状态边界梳理。
  - 阶段 1：编辑后即时反馈。
  - 阶段 2：图层版本与瓦片缓存。
  - 阶段 3：脏瓦片精确刷新。
  - 阶段 4：多尺度概化表。
  - 阶段 5：离线瓦片发布。
  - 阶段 6：并发编辑与冲突处理。
- 按 Agent A-E 拆分了可交给 Codex agent 的任务包。
