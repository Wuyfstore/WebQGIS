# 大数据矢量编辑优化方案细化

## 本次目标

根据 `进度文档/20260617_181334_高亮路径与大数据渲染优化方案.md` 中关于大数据表显示优化的思路，进一步补全 WebQGIS 场景下“既保证浏览速度，又保证矢量编辑后快速响应改变”的落地方案。

重点不是只做大数据浏览，而是明确 WebQGIS 需要支持 PostGIS 矢量数据的选择、编辑、保存和即时反馈。

## 完成内容

- 重写并细化 `设计文档/大数据矢量编辑性能优化实施方案.md`。
- 明确采用“浏览层 MVT/缓存瓦片 + 选择高亮层 GeoJSON + 编辑覆盖层 GeoJSON + tileVersion/dirtyTiles 刷新协调”的架构。
- 补充为什么不推荐全量 GeoJSON 加载，也不推荐只依赖离线瓦片。
- 将实施拆成 7 个阶段：
  - 阶段 0：状态边界梳理。
  - 阶段 1：编辑后即时反馈。
  - 阶段 2：tileVersion 与瓦片缓存。
  - 阶段 3：dirtyTiles 精确刷新。
  - 阶段 4：多尺度概化表。
  - 阶段 5：离线瓦片发布。
  - 阶段 6：并发编辑与冲突处理。
- 每个阶段补充了：
  - 目标。
  - 要解决的问题。
  - 实现边界。
  - 建议改动文件。
  - 具体实现步骤。
  - 验收标准。
  - 测试建议。
  - 可直接复制给 Codex agent 的任务描述。
- 增加了任务拆分总表和每个 Codex agent 的通用执行要求。
- 增加了最小可交付路线，方便先完成可用闭环。
