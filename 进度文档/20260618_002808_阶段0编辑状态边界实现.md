# 阶段0编辑状态边界实现

## 本次目标

按照 `设计文档/大数据矢量编辑性能优化实施方案.md` 的阶段 0，梳理 WebQGIS 前端“浏览、选择、编辑”三类状态边界，解决点击选择、范围选择、自定义范围选择、图层切换和编辑草稿之间互相覆盖的问题。

本次使用了多 agent 协同做只读审查：

- 一个 agent 审查状态边界和静默丢草稿风险。
- 一个 agent 审查阶段 0 的测试覆盖方式。

## 做了哪些

- 在 `useWebGisWorkspace` 中增加编辑草稿 dirty 状态：
  - `isDraftDirty`
  - `hasUnsavedEditDraft`
  - `markDraftDirty()`
  - `markDraftClean()`
  - `clearSelectedFeatureState()`
- 将“清选择元信息”和“清整个编辑草稿”拆开，避免范围选择只想清旧高亮时把整个编辑状态误清。
- 在 `useOpenLayersEditor` 中增加选择/编辑视觉清理能力：
  - `clearSelectionFeatures()`
  - `clearEditableFeature()`
- 绘制结束和节点修改结束后会标记草稿 dirty。
- 点击地图读取其他要素前会调用 `canDiscardDraft`，脏草稿存在时不再静默替换当前编辑 Feature。
- 工作台切换到范围选择或自定义范围选择前会检查脏草稿：
  - 有脏草稿：阻止切换并提示保存或清除。
  - 无脏草稿：清理旧选择高亮和干净的编辑视觉。
- 切回点击选择时清理范围选择高亮，但不清理编辑草稿。
- 图层选择、刷新图层、缩放到图层、独显图层、恢复可见性、打开样式编辑、加载图层、移除图层、关闭编辑等入口增加草稿保护。
- `activeLayerId` watcher 在发现外部路径切换且存在脏草稿时，会尽量回滚到原活动图层，避免“编辑图层”和“活动图层”分裂。
- `EditInspector` 属性输入会发出 `propertyChange`，工作台收到后标记草稿 dirty，避免只改属性时被当成干净草稿。
- 补充 Vitest 覆盖：
  - 切到范围/自定义范围前清理旧选择和编辑视觉。
  - 脏草稿阻止切换到范围选择。
  - 属性编辑会让草稿变脏并阻止切换选择方式。
  - 脏草稿阻止缩放到其他图层和打开样式编辑继续执行。
  - 切回点击选择时清理范围高亮但不清编辑源。

## 没做哪些

- 未实现阶段 1 的“保存后即时覆盖显示”。
- 未实现 MVT covered feature id 隐藏/弱化。
- 未实现 tileVersion、dirtyTiles、瓦片缓存、多尺度概化表或离线瓦片发布。
- 未做真实 PostGIS 数据的浏览器端要素点击保存验证；Chrome 只做本地页面渲染冒烟。

## 哪里有 bug

- 本次未发现新的测试失败。
- 仍有一个产品层面的后续风险：`workspace.refreshAll()` / 数据源扫描导致已加载活动图层从目录中消失时，workspace 内部仍可能清理草稿；这类数据源变化场景需要阶段 1 或后续状态模型继续收敛。
- 当前 `hasUnsavedEditDraft` 仍要求存在 `draftGeometry` 且 dirty。只改属性时，本次通过属性输入事件配合已有几何来保护；如果未来支持“先填属性、后绘几何”的新建流程，需要进一步扩展草稿模型。
