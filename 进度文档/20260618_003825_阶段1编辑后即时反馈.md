# 阶段1编辑后即时反馈

## 本次目标

按照 `设计文档/大数据矢量编辑性能优化实施方案.md` 的阶段 1，完成矢量要素保存后的即时反馈能力：保存接口返回服务端确认后的完整 Feature 后，前端立即用该 Feature 更新编辑覆盖层和工作区状态，并临时遮盖同图层同主键的 MVT 旧要素，避免用户在瓦片刷新前看到旧数据。

## 做了哪些

- 在 OpenLayers 编辑器中增加按图层维护的 `coveredFeatureIdsByLayer`，用于记录已被编辑覆盖层替代显示的 MVT 要素主键。
- 新增 `isFeatureCovered` 纯函数，用于判断某个 MVT feature 是否应被隐藏，并补充单元测试。
- 在图层样式函数中对被覆盖的 MVT 要素返回空样式，使保存后的最新 Feature 由编辑覆盖层优先展示。
- 暴露 `markFeatureCovered`、`clearCoveredFeature`、`clearCoveredFeatures`，便于后续阶段在瓦片版本刷新、撤销或关闭编辑时回收覆盖记录。
- 保存要素成功后使用服务端返回的完整 Feature 调用 `editor.loadEditableFeature(saved)`，并标记对应图层和要素为 covered，再刷新图层样式。
- 工作区保存成功后显式把 `draftGeometry` 更新为服务端返回的 geometry，保证属性面板、编辑草稿和地图覆盖层对齐。
- 补充工作台组件测试，覆盖保存成功时即时使用服务端 Feature、标记 MVT cover、刷新图层，以及保存失败时不更新覆盖层、不隐藏 MVT 旧要素。

## 没做哪些

- 未实现 `tileVersion`、`dirtyTiles` 或瓦片级增量刷新，这属于后续阶段 2/3。
- 未做真实 PostGIS 写入后的端到端高亮验证，本次 Chrome 只做页面加载和 console error 冒烟检查。
- 未新增独立的 `useWebGisWorkspace` 单元测试文件；当前通过工作台组件测试覆盖主要保存链路。

## 哪里有 bug / 风险

- `coveredFeatureIdsByLayer` 目前不会因为瓦片版本刷新自动清理，后续实现 tileVersion/dirtyTiles 后需要在新瓦片确认加载完成时回收对应 covered 记录。
- 当前覆盖策略按图层和要素主键隐藏旧 MVT，要确保后续新增要素、复制要素或服务端改写主键时继续使用服务端返回的最终 id。
- Chrome 冒烟验证结果：页面标题为 `WebQGIS`、URL 为 `http://127.0.0.1:5173/`、console error 为空；DOM 快照未命中字面量 `WebQGIS`，不影响页面标题级验证，但不代表业务链路已完整验证。

## 建议后续做什么

- 阶段 2 增加保存返回的 tileVersion/dirtyTiles 元数据，并建立前端脏瓦片索引。
- 阶段 3 在局部瓦片刷新完成后清理 covered 记录，让临时遮盖状态随服务端瓦片版本收敛。
- 阶段 4 以后再加入真实浏览器 E2E：选择要素、编辑属性或几何、保存、确认覆盖层即时变化、瓦片刷新后覆盖记录消失。
