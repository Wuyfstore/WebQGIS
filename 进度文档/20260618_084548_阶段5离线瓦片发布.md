# 阶段5离线瓦片发布

## 本次目标

实现“大数据矢量编辑性能优化实施方案”阶段 5 的最小可落地闭环：为稳定或半稳定大数据图层提供目录型离线 MVT 发布能力，让浏览层可优先读取已发布瓦片，缺失或不可读时自动回退 live PostGIS MVT，同时保持编辑保存后仍通过编辑覆盖层即时显示。

## 做了哪些

- 定义离线瓦片元数据：
  - 新增 `TileSourceType`。
  - 新增 `TilePackage`。
  - 在前后端 `LayerRegistration` 中增加 `tileSourceType` 与 `tilePackages`。
- 新增后端 `tile-cache` 模块：
  - `TilePackageRepository` 从 `data/tile-packages.json` 读取发布包元数据。
  - `TileCacheService` 支持按 layer/zoom 选择匹配包，并读取目录结构 `{z}/{x}/{y}.mvt`。
  - 相对 `storagePath` 限制在 `data/tile-packages` 根目录下，避免路径逃逸。
- 修改 `LayersService.getVectorTile()`：
  - `tileSourceType` 为 `directory/pmtiles/mbtiles` 时先查找发布包。
  - 目录瓦片命中时直接返回离线 MVT。
  - 离线包缺失、zoom 不匹配或瓦片文件缺失时回退 live PostGIS MVT。
  - 内存缓存 key 纳入 package id/version，避免发布包更新后串用旧 Buffer。
- 修改图层仓储：
  - 数据源重新扫描替换图层时保留 `tileSourceType`、`tilePackages`、`scaleSources` 和 `tileVersion`。
  - 旧图层默认 `tileSourceType` 为 `live`。
- 新增 API 路由：
  - `/api/layers/:id/offline-tile/:z/:x/:y.mvt`
  - 该路由复用同一个 service 逻辑，因此仍具备 live 回退能力。
- 修改前端瓦片 URL 派生：
  - `live/cached` 继续使用原 `tileUrl?v=tileVersion`。
  - `directory/pmtiles/mbtiles` 使用 `/offline-tile/` API 路径。
  - 当前前端不直接解析 PMTiles/MBTiles 文件，统一交给 API 做分发与回退。
- 新增瓦片发布脚本：
  - `scripts/generate-tiles.mjs`
  - dry-run 默认输出发布计划、瓦片数量和样例。
  - `--execute` 时调用现有 live MVT API 下载并写入目录瓦片。

## 没做哪些

- 没有实现 PMTiles 浏览器直读协议，也没有引入 PMTiles 客户端依赖。
- 没有实现 MBTiles SQLite 读取；当前只完整支持目录型 MVT 包。
- 没有实现后台队列、任务状态、失败重试和自动重建 dirtyTiles。
- 没有新增前端 UI 来编辑 `tileSourceType` 或管理 `tilePackages`；当前依赖图层元数据配置。
- 没有在真实大数据集上执行完整离线发布压测；本轮验证覆盖代码路径和 dry-run 发布计划。

## 哪里有 bug

- 本轮未发现新的阻塞 bug。
- 当前 `pmtiles/mbtiles` 类型只是为元数据和后续扩展预留；实际读取仍会因为没有目录包实现而回退 live MVT。
- 目录包是否“最新”当前只按 `version` 降序选择，尚未引入 `ready/building/failed` 状态，后续后台发布任务需要补状态字段。
- `scripts/generate-tiles.mjs --execute` 依赖 API 服务可访问；如果 API 未启动或图层 live MVT 本身不可用，发布会失败。

## 建议后续做什么

- 增加 tile package 管理 API：
  - 创建发布任务。
  - 查询发布状态。
  - 切换活动发布包。
  - 删除旧发布包。
- 引入 `status`、`errorMessage`、`tileCount`、`completedAt` 等发布包字段，避免使用未完成包。
- 实现 dirtyTiles 后台重算：
  - 小范围编辑后只重建受影响瓦片。
  - 批量更新后按 zoom 范围重建。
  - 发布完成后刷新 package version。
- 如确实需要 PMTiles/MBTiles：
  - PMTiles 可优先用于静态文件/CDN 分发。
  - MBTiles 可在服务端用 SQLite 读取并复用当前 `/offline-tile/` API。
- 增加真实数据压测，记录 live MVT、目录离线 MVT 和多尺度概化表三种路径的耗时差异。
